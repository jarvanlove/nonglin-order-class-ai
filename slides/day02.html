1. 生成（Genetic）

Agent 根据当前任务，生成多个不同版本的提示词变体。每个变体像一条"DNA"，包含不同的表达策略、思考步骤、输出格式。

2. 评估（Pareto）

每个变体执行同一批测试任务，按准确率、速度、资源消耗、稳定性评估。使用帕累托最优筛选：不存在另一个变体在所有指标上都比它好。

3. 进化（Evolution）

优秀变体保留，进行"交叉"和"变异"，组合成新变体。循环回到 Step 1，持续迭代。

实际效果：代码生成准确率提升 23%，冗余步骤减少 40%。进化过程完全自动化。

工作记忆（Working Memory）

当前对话的短期上下文，读写速度极快，断电即失。类比人脑的前额叶。

情景记忆（Episodic Memory）

存储过去的任务执行记录和结果。通过语义相似度检索："我之前做过类似的事吗？"

程序记忆（Procedural Memory）

存储经过验证的最佳实践和技能模板。GEPA 进化出的优秀提示词，跨任务复用。

对比总结

生态优先 vs 进化优先

OpenClaw
开源 Agent 的"安卓系统"
  • ✓ 33 万 Stars，生态极度成熟
  • ✓ 13,700+ Skills 即装即用
  • ✓ 20+ 渠道统一接入
  • ✓ 适合重复性工作流
  • ✓ 适合多系统集成
Hermes Agent
自我进化的"特斯拉"
  • ✓ 10 万 Stars，增长极快
  • ✓ GEPA 提示词自进化
  • ✓ 400+ 模型支持
  • ✓ 适合复杂推理任务
  • ✓ 适合需要经验积累的场景
两者可结合：OpenClaw 做调度，把复杂任务丢给 Hermes Agent 处理。选型看场景、看团队、看部署环境。
