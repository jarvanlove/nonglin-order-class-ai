Day 5 · Trae 调试三板斧
Debug

Trae 调试三板斧

1. 终端错误（Terminal）

  • 从下往上读堆栈，最后一行是线索
  • 常见格式：Error: xxx at line 15
  • 直接复制报错给 Builder，让 AI 解读

2. 浏览器控制台（F12）

  • 看红色报错、看黄色警告
  • Network 标签查状态码 404 / 500
  • 前端项目必开，定位渲染问题

3. Problems 面板

  • 红色波浪线 = 必须修
  • 黄色波浪线 = 建议修
  • 语法错误提前发现，别忽视
💡 核心技巧：报错直接复制给 Builder，让 AI 帮你解读并修复——不要自己硬啃。

Day 5 · 常见 Builder 报错速查（上）
Troubleshooting

常见 Builder 报错速查（上）

报错信息原因解决方法
Failed to fetch API 地址错误或网络不通 检查地址拼写、查看 Network 状态码、确认 CORS
API Key invalid Key 未填、填错或已过期 确认 Key 完整无空格、区分开发/生产环境
Cannot read property of undefined 数据结构不对，访问了不存在的属性 先 console.log(data) 看返回结构，再调整代码
页面空白 JS 执行出错或 DOM 元素未获取 F12 看 Console 红色报错、检查元素 ID 是否匹配
🔍 先 console.log 再看报错，定位效率翻倍
💬 把报错直接贴给 Builder，AI 会帮你修复

Troubleshooting

常见 Builder 报错速查（下）

报错信息原因解决方法
Module not found import 路径错误或依赖缺失 检查路径、改用原生 JS、确认 CDN 引入
样式不生效 CSS 未引入或选择器不匹配 检查 link 标签、F12 Elements 看实际样式
Builder 一直"思考中" Prompt 太长或 Trae 繁忙 简化 Prompt，一次一个需求，必要时取消重发

Day 5 · 测试三级跳
Testing

测试三级跳

第一级

功能测试

  • 每个功能点至少测一次
  • 输入框测：正常 / 空值 / 特殊字符
  • 列出所有功能点逐一打勾
第二级

集成测试

  • 三模块协同走完整流程
  • 异常流程也要测（空数据、格式错误）
  • 验证数据在各模块间传递完整
第三级

用户验收测试（UAT）

  • 找另一组当用户，无指导操作
  • 3 分钟无指导完成 = 优秀
  • 记录困惑点和操作卡点
测试不是找茬，是确保明天路演不翻车

Day 5 · Bug 修复优先级
Bug Triage

Bug 修复优先级

P0 · 阻断性

核心功能不可用 / 程序崩溃

  • 页面打开空白
  • 点击核心按钮无反应
  • 数据读取失败导致流程中断
立即修，15 分钟修不好就降级（用静态数据代替）
P1 · 功能性

结果不对 / 部分场景失效

  • 筛选功能有时有效有时无效
  • 数据显示不完整
  • 搜索对中文支持不好
尽量修，太复杂就记为"已知问题"
P2 · 体验性

颜色不好看 / 加载稍慢

  • 按钮颜色不美观
  • 页面加载慢（但可接受）
  • 提示文字不够友好
有余力再修，文档注明"已知优化点"
原则：不要在小问题上纠结半小时，先保主流程

Day 5 · 项目文档模板展示
Documentation

项目文档模板展示

README 标准结构

  1. 项目简介 — 一句话说明做什么
  2. 功能特性 — 列出核心功能点
  3. 技术方案 — 三工具分工表
  4. 安装运行 — 环境要求 + 步骤
  5. 项目结构 — 文件树说明
  6. 使用说明 — 一步一动作
  7. 团队成员 — 姓名 + 职责 + 模块

技术方案文档

  1. 需求背景 — 为什么做这个项目
  2. 总体架构 — 三工具如何协作
  3. 模块设计 — 输入 / 处理 / 输出
  4. 关键决策 — 选型理由
  5. 测试方案 — 测了什么，结果如何
  6. 风险评估 — 风险 + 应对措施
📝 文档是写给人看的，假设读者完全不懂技术。一步一动作、配截图、标预期结果、含故障排除。
