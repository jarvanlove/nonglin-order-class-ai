常见阻塞 Top 8（上）

QClaw 微信连接失败

先查微信版本，再切手机热点，最后重扫码。90% 是网络或版本问题。

🔐

WorkBuddy 文件权限不足

右键"以管理员身份运行"，检查隐私设置，避免放到 C 盘根目录。

💻

Trae Builder 报错

缺依赖就 npm install，端口占用就 kill-port，报错丢给 Chat 模式修。

📄

数据格式不匹配

QClaw 明确指定 JSON 字段，WorkBuddy 配置字段映射，确保 UTF-8 编码。

1

去 ClawHub 搜索

按场景筛选，优先选使用人数多、评分高的 Skill

→
2

点击"添加"

像装微信小程序一样，先安装才能调用

→
3

@Skill名 调用

对话中直接 @Skill名 加参数，观察结构化输出

学生任务 为项目选一个 Skill，完成一次实际采集，保存到桌面 qclaw-output.json

Day 4 · 联调流程
下午 15:45 - 17:30

联调五步法：QClaw → WorkBuddy 数据流

1

确认数据出口

QClaw 侧输出文件正常，内容合法 JSON，编码 UTF-8

→
2

确认数据入口

WorkBuddy 能读取并预览数据，字段名和记录数正确

→
3

设计转换规则

清洗、重组、格式化：去重、空值处理、日期统一

→
4

端到端测试

从采集到报告全程自动，抽查 2-3 条数据对比准确性

→
5

边界测试

空数据、缺失字段、特殊字符、大数据量逐一验证

核心原则：两个工具之间不会自动"心有灵犀"，你必须当翻译——告诉 QClaw 按什么格式输出，告诉 WorkBuddy 按什么格式读取。

数据接口约定：QClaw ↔ WorkBuddy

{
  "version": "1.0",
  "generated_by": "QClaw",
  "generated_at": "2026-04-27T14:30:00+08:00",
  "data": [
    {
      "id": 1,
      "title": "信息标题",
      "content": "信息正文或摘要",
      "source": "来源网址或文件名",
      "category": "分类标签",
      "date": "2026-04-27",
      "confidence": 0.95
    }
  ]
}

文件位置

桌面 project-data/
qclaw-output.json

编码格式

UTF-8
禁止 ANSI / GBK

字段命名

英文小写 + 下划线
禁止中文字段名

空值处理

用 "" 空字符串
不用 null
