今日目标

Day 5 学习目标：让项目"有界面、有逻辑、有数据"

知

知识目标

  • 理解 API 的概念：API 是不同软件之间的"对话方式"，让项目能获取外部实时数据
  • 掌握 JSON 数据格式：学会看懂和编写简单的 JSON 结构
  • 理解调试的基本原理：Bug 是程序中的错误，调试是定位并修复的过程
  • 了解三工具整合的数据流：QClaw 采集 → WorkBuddy 清洗 → Trae 展示

核心认知：API 让静态网页变成"活"的应用，调试是编程的必备技能

技

技能目标

  • 掌握 Trae 进阶技巧：Chat 模式与 Builder 模式的灵活切换
  • 能在 Trae 中调用公开 API（如天气、新闻），并将数据显示到页面上
  • 掌握调试三板斧：看终端报错、检查文件路径、确认数据格式匹配
  • 完成 QClaw + WorkBuddy + Trae 三工具的完整整合联调

核心技能：从"会用工具"到"会修问题"，从"单工具操作"到"跨工具整合"

养

素养目标

  • 培养工程思维：面对报错不慌，系统性地排查问题
  • 建立问题排查能力：从下往上读报错、对比预期与实际输出
  • 养成文档习惯：项目完成后撰写 README，让代码可被他人理解
  • 强化团队协作：明确模块接口约定，减少联调时的摩擦

核心素养：好的工程师不是不写 Bug，而是能快速定位和修复 Bug
