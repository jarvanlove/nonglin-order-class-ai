Day 05 · Agentic Engineering

Agentic Engineering

Day 5 项目作战图：不是重复演进史，而是进入项目执行

今天的核心变化

  • Day 3 看演进：理解 Vibe Coding 为什么出现
  • Day 5 做项目：把 AI 当协作团队来组织
  • 从单次提示词，升级为目标、角色、工具、验收
  • 从效果好不好看，升级为能否交付、能否复盘

项目指挥官要做什么

  • 定目标：只做 1 个用户、1 条主流程
  • 拆任务：谁负责内容、页面、测试、路演
  • 控边界：不做登录、后端、真实 API
  • 收证据：工作流、版本、Bug、截图、知识卡
  • 做决策：AI 给方案，人判断取舍

课堂判断

  • 能不能说清用户和场景
  • 能不能把任务交给不同 AI 工具
  • 能不能测试主流程是否能跑
  • 能不能记录 Bug 和修复过程
  • 能不能讲清个人贡献
Planner需求卡 / 主流程 / 验收标准
→
Builder页面 / 文档 / PPT / 版本截图
→
Tester测试清单 / Bug 记录 / 复测结论
→
Speaker路演稿 / 简历项目 / 问答准备

必须留下的证据

  • 需求卡：为什么做这个
  • 工作流图：每一步谁负责
  • 测试清单：怎么证明能跑
  • Bug 记录：怎么证明迭代过
  • LLM-Wiki：怎么沉淀复用

讲师过渡话术

  • 这不是再看一遍演进图
  • 从这一页开始，每个小组就是一个 AI 项目团队
  • AI 可以干活，但项目经理必须是人
  • 今天看的是交付证据，不是口头说会用 AI

Day 05 · 理论 + 实操

理论 + 实操

课堂示例：为什么综合项目不能只靠一句提示词

必讲理论

  • 综合项目包含资料、内容、页面、测试、展示
  • 一个 AI 回答无法同时保证所有环节正确
  • Agentic 的价值是分工、交接和验收
  • 人的角色是项目负责人，不是旁观者

教师现场演示

  • 把“做一个求职助手”拆成 5 步
  • 演示 Builder/Tester/Reviewer 三个角色
  • 展示每一步的输入输出

学生互动

  • 小组给自己的项目分配角色
  • 写出每个角色的交付物

课堂产出

  • Agentic 工作流图

讲师判断标准

  • 能不能复述方法
  • 能不能独立改提示词
  • 能不能指出 AI 输出的问题
  • 能不能把结果变成自己的作品

Day 05 · 案例 10 操作手册

案例 10 操作手册

综合项目规划：课堂怎么带着做

工具与课前准备

  • 使用工具：QClaw + WorkBuddy + Trae + 文档 AI
  • 准备项目模板
  • 三选一：AI 求职助手、学习资料整理助手、校园活动助手
  • 统一不做登录、后端、真实 API

教师演示步骤

  1. 选题并砍范围
  2. 用 QClaw 搜集资料
  3. 用 WorkBuddy 整理字段
  4. 用 Trae 生成 Demo
  5. 设置 Tester 找 Bug

学生跟做步骤

  1. 选题
  2. 分角色
  3. 搜集资料
  4. 生成 Demo
  5. 测试并记录

可直接投屏的提示词

我们要在 1 天内完成【项目】MVP。请输出工作流表：步骤、负责角色、使用工具、输入、输出、验收标准、风险。限制：不做后端、不接真实 API、必须能课堂演示。

常见问题与处理

问题处理办法
项目过大砍到单页 MVP
角色重叠明确 Planner/Builder/Tester/Speaker
没有验收每一步写可检查标准

课堂检查点

能复述任务能独立操作能发现问题能提交产出

Day 05 · 案例 12 操作手册

案例 12 操作手册

LLM-Wiki 个人知识卡：课堂怎么带着做

工具与课前准备

  • 使用工具：本地 LLM-WIKI / Markdown / 文档 AI
  • 准备知识卡模板
  • 字段：主题、问题、做法、证据、下次复用
  • 要求每人沉淀 1 张卡

教师演示步骤

  1. 展示 LLM-Wiki 的用途
  2. 把一个 Bug 修复记录变成知识卡
  3. 把一个提示词原则变成知识卡
  4. 说明链接关系

学生跟做步骤

  1. 选择一个今天的问题
  2. 写知识卡
  3. 链接到项目
  4. 写下次复用场景

可直接投屏的提示词

请把下面项目经历整理成一张知识卡：主题、遇到的问题、我怎么判断、AI 帮了什么、人做了什么决定、下次如何复用。

常见问题与处理

问题处理办法
写成日记改成问题-做法-复用
只写结果补过程和证据
没有链接链接到项目、Bug 或提示词原则

课堂检查点

能复述任务能独立操作能发现问题能提交产出
