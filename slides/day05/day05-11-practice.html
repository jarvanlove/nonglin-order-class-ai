课堂实操指导：API 接入、调试演练、文档填写

API 接入步骤（助教演示 + 学生跟做）

1

打开 Trae，进入 Builder 模式

确认左侧文件树显示项目目录，新建一个 api-demo.html 文件。

2

输入 API 调用提示词

"创建一个网页，调用天气 API 显示北京今天的天气，包含温度、天气状况、湿度。"

3

审阅 Plan 并确认执行

检查 AI 生成的计划是否合理，确认后点击执行。

4

用 Live Server 预览

右键 HTML 文件 → "Open with Live Server"，在浏览器中查看效果。

5

遇到报错？复制到 Chat 面板

按 F12 打开控制台，复制报错信息，在 Chat 中说"这个报错怎么解决"。

文档模板填写指南

项目简介 一句话：我们做了一个 XX 系统，解决 YY 问题
功能特性 列出 3-5 个功能，如"自动采集农业政策""数据清洗与分类""网页展示与搜索"
技术方案 说明用了哪三个工具，各自负责什么模块
安装步骤 1. 安装 Trae 2. 打开项目文件夹 3. 用 Live Server 运行
团队成员 姓名 + 负责模块 + 联系方式

助教巡视要点

1. 每组至少完成一次 API 调用并成功显示数据

2. 检查字段名是否统一（QClaw/WorkBuddy/Trae 三方对照）

3. 确认数据文件使用 UTF-8 编码

4. 帮助落后组简化需求，先完成基础功能
