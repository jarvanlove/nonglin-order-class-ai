Trae 进阶技巧：Chat 与 Builder 模式切换 + 代码审查

两种工作模式，两种使用场景

Trae 提供 Chat 模式和 Builder 模式，理解它们的区别能让你在不同场景下选择最高效的方式。

Chat 模式

  • 适合：修改已有代码、解答疑问、局部优化
  • 操作：选中代码 → 右键"询问 AI" → 描述需求
  • 优势：精准、可控、不会大面积重写
  • 示例："帮我把这个函数改成异步的"

Builder 模式

  • 适合：从零创建项目、添加新功能模块
  • 操作：描述需求 → AI 生成 Plan → 确认执行
  • 优势：全自动、生成完整文件结构
  • 示例："创建一个天气展示网页，调用公开 API"

切换策略建议

上午用 Builder 模式搭建页面骨架，下午用 Chat 模式微调细节和修复 Bug。不要全程只用一种模式——Builder 适合"从 0 到 1"，Chat 适合"从 1 到 1.1"。

代码审查：让 AI 帮你找问题

写完代码后，让 Trae 帮你审查一遍，往往能发现你自己忽略的问题。

// 在 Chat 面板输入以下提示词：

请审查这段代码，检查：

1. 是否有语法错误

2. 变量名是否清晰

3. 是否有潜在的空值风险

4. 是否有性能优化空间

// 然后粘贴你的代码

多文件项目管理技巧

文件导航：按 Ctrl+P 快速打开文件，按 Ctrl+Shift+F 全局搜索。

分屏编辑：右键文件标签 → "向右拆分"，同时查看 HTML 和 CSS。

跨文件修改：在 Chat 模式中说"帮我把所有页面的标题统一改成 XXX"，AI 会自动找到所有相关文件。

常见误区

误区 1：Builder 模式生成的代码不需要检查。事实：AI 也会犯错，尤其是路径和变量名。

误区 2：Chat 模式只能问代码问题。事实：可以问"怎么设计这个页面的布局更好"等非代码问题。
