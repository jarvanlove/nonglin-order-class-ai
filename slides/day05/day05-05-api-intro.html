API 基础

API 入门：让项目获取"活"数据

API 是什么？

API（Application Programming Interface，应用程序接口）可以理解为"不同软件之间的对话方式"。就像农机调度系统：你在线提交需求（发送请求），系统返回可用农机和预约状态（返回数据）。你不需要知道后台怎么调度，只需要会"提交需求"就行。

通俗例子：农业气象 API

你想在网页上显示"今天适合喷药吗"，但你的网页不会"看天"。于是你调用一个农业气象 API：

你问："今天温度、湿度和风速怎么样？"

API 答："温度 25°C，湿度 65%，风速 3 级，建议下午喷药"

你的网页只需要把 API 返回的数据显示出来，帮助农户判断农事操作时机。

Trae 如何调用 API

在 Trae 的 Builder 或 Chat 模式中，你可以用自然语言描述需求，AI 会自动生成调用 API 的代码。核心步骤：

1. 找到公开 API（如农业气象、农产品价格、土壤墒情）

2. 在 Trae 中描述："调用 XX API，获取数据并显示在页面上"

3. AI 生成 fetch/axios 代码，你只需确认执行

JSON 数据格式简介

API 返回的数据通常是 JSON 格式——一种人类和机器都能读懂的结构化文本。想象它是一个"带标签的盒子"，每个数据都有名字。

// 农业气象 API 返回的 JSON 示例

{

"region": "华北平原",

"temperature": 25,

"humidity": 65,

"windSpeed": 3

}

字段名
含义
region
农业区域名称
temperature
温度（摄氏度）
humidity
湿度百分比
windSpeed
风速等级

注意事项

公开 API 通常有调用频率限制（如每分钟 60 次），超过限制会被暂时封禁。开发时建议先用少量数据测试，不要频繁刷新页面。
