整合测试 Checklist

  • QClaw 能正常采集并输出 JSON 文件到约定位置
  • WorkBuddy 能读取 QClaw 的输出文件，无乱码
  • WorkBuddy 清洗后的数据包含所有约定字段
  • Trae 页面能读取 WorkBuddy 的输出文件
  • 页面上显示的数据无 undefined、无空白
  • 数据文件缺失时，页面显示友好提示而非报错
  • 从 QClaw 重新采集 → WorkBuddy 处理 → Trae 刷新，全流程 < 5 分钟
  • 各模块使用统一的字段名（英文小写 + 下划线）
  • 所有文件使用 UTF-8 编码，无中文乱码
  • 至少完成一轮端到端测试，记录测试结果

联调失败常见原因

1. 字段名不一致：QClaw 用 "title"，Trae 代码里用 "name"

2. 文件路径错误：相对路径起点搞错，或文件夹未创建

3. 编码问题：文件不是 UTF-8，导致中文显示为乱码

4. 数据格式错误：JSON 外层包裹了 Markdown 代码块标记
