问题排查

调试技巧：Bug 不可怕，可怕的是不会找

Bug 是什么？

Bug 是程序中的错误。写代码不出错是不可能的——即使是资深工程师，每天也会遇到各种报错。关键不是避免 Bug，而是快速定位和修复。

调试三板斧

第一斧：看终端报错（从下往上读）

报错信息通常很长，但真正的错误原因在最下面几行。从下往上读，找到 "Error:" 或 "Uncaught" 开头的行。

第二斧：检查文件路径

80% 的 "找不到文件" 错误都是路径问题。确认：文件名拼写正确、大小写一致、相对路径起点正确（以 HTML 文件所在位置为起点）。

第三斧：确认数据格式匹配

API 返回 JSON，但代码按字符串处理？字段名是 "title" 但代码里写 "name"？对比预期和实际的数据结构。
