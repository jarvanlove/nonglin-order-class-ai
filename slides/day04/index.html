Day 04 · 理论 + 实操

理论 + 实操

课堂示例：为什么“帮我做网页”会失败

必讲理论

  • 需求不清，AI 只能猜页面结构
  • 没有约束，AI 可能生成多文件、后端、复杂依赖
  • 没有验收，学生不知道算不算完成
  • 不会代码也能当产品经理和测试员

教师现场演示

  • 先输入一句“帮我做活动页”
  • 展示失败点
  • 改成六字段需求
  • 再生成并对比

学生互动

  • 学生把自己的 Day 3 作品改成六字段需求

课堂产出

  • 需求说明书 v1

讲师判断标准

  • 能不能复述方法
  • 能不能独立改提示词
  • 能不能指出 AI 输出的问题
  • 能不能把结果变成自己的作品

可直接投屏的提示词

请基于以下活动文案和视觉风格，生成单文件 HTML 活动页。功能：展示活动信息、3 个亮点、报名表单、点击提交后显示报名成功和活动提醒。不需要后端。

常见问题与处理

问题处理办法
生成太多文件补充单文件 HTML
按钮无反馈检查表单 submit/click 事件
报名数据不会保存说明无后端限制，只做课堂演示

提示词评分器：课堂怎么带着做

工具与课前准备

  • 使用工具：Trae / 浏览器 / 文档 AI
  • 准备评分维度：目标、上下文、格式、约束、验收

  • 要求学生做一个可交互工具

教师演示步骤

  1. 展示评分标准
  2. 让 AI 生成评分器页面
  3. 输入烂提示词测试

  5. 修复显示问题

学生跟做步骤

  1. 生成评分器
  2. 输入自己的提示词
  3. 查看评分和建议
  4. 修改提示词再测一次

可直接投屏的提示词

请做一个单文件 HTML 提示词评分器。用户粘贴提示词后，点击评分，页面按 5 个维度给分：目标、上下文、格式、约束、验收，并给出修改建议。无需调用真实 API，用前端规则模拟即可。

常见问题与处理

问题处理办法
学生以为要接 API明确今天用规则模拟
评分不准确强调这是学习工具，评分规则可人工调整
JS 报错复制控制台错误让 Trae 修

课堂检查点

能复述任务能独立操作能发现问题能提交产出

案例 9 操作手册

Bug 修理铺：课堂怎么带着做

工具与课前准备

  • 使用工具：Trae / Chrome 控制台
  • 准备故障版本页面
  • 故障包括按钮无反应、undefined、文字溢出
  • 准备 Bug 记录模板

教师演示步骤

  1. 复现 Bug
  2. 描述：操作-现象-期望
  3. 复制控制台错误
  4. 让 AI 一次只修一个
  5. 回归测试

学生跟做步骤

  1. 列 3 个 Bug
  2. 按优先级修
  3. 每修一次测试
  4. 记录修改前后

可直接投屏的提示词

点击【按钮】后没有任何反应。请检查事件绑定和相关 JavaScript。只修复这个问题，不要重写页面，不要改变视觉样式。修复后说明改了哪里。

常见问题与处理

问题处理办法
AI 重写全页加限制：不要重写，只改指定区域
修完又坏回退到上一版本，重新只修一个问题
学生不会看控制台讲师示范 F12/Console

课堂检查点

能复述任务能独立操作能发现问题能提交产出

Day 04 · Debug 理论

Debug 理论

修 Bug 的四步：复现、定位、修复、回归

步骤学生要写什么示例
复现我做了什么，出现什么点击提交后没有成功提示
定位大概是哪类问题按钮事件可能没有绑定
修复一次只改一处只检查 submit/click 事件
回归修完再测旧功能输入为空、正常输入、重复点击都试一次
