Day 4 · 知识储备

进阶技巧

QClaw 进阶：让信息搜集更精准

6 大进阶技巧

1. 多轮对话优化

先给大方向看输出，再逐步细化，不要一次写太长的 Prompt

2. 输出格式控制

明确告诉 QClaw 你要的格式：表格、bullet points、字数限制

3. 来源验证

让 QClaw 提供来源链接，你抽查 2-3 个验证真实性

4. 批量处理

写模板化 Prompt："按照以下格式整理 [主题] 的资料"

5. 角色设定

在 Prompt 开头给 QClaw 设定角色："你是一位农业技术专家..."

6. 条件筛选

添加筛选条件："优先选择有具体数据支撑的案例"

互动：Prompt 优化比赛（3 分钟）

基础 Prompt："搜集农业 AI 案例"

请各组在 3 分钟内优化这个 Prompt，全班投票选出"最佳 Prompt"。

讲师点评要点

哪些优化是有效的（具体、有格式、有筛选）？哪些是多余的（过于冗长、限制过死）？

进阶 Prompt 模板

【角色】

你是一位 [领域] 专家

【任务】

请搜集 [时间范围] 内 [主题] 的信息

【筛选条件】

1. 优先选择 [条件]
2. 排除 [条件]
3. 只选择 [条件]

【要求】

至少 [数量] 条记录，按 [维度] 分类，每个案例包含 [字段列表]，输出格式 [格式]，提供来源链接

【示例】

给出一个期望的输出样例

常见误区

  • "Prompt 很长但输出差" → 先给大方向，再逐步细化
  • "来源链接打不开" → AI 会"幻觉"，抽查 2-3 个即可
  • "资料和其他组重复" → 按不同角度分类，产出完全不同

Day 4 · 知识储备

进阶技巧

WorkBuddy 进阶：让数据处理更高效

5 大进阶技巧

1. 批量文件处理

把多个同类型文件放在一个文件夹，WorkBuddy 可以批量处理

2. 自动化流程

把常用操作保存为"流程"，下次一键执行

3. 数据格式转换

Excel ↔ CSV ↔ JSON ↔ Markdown，确保 QClaw 输出能被 Trae 读取

4. 条件判断

"如果金额大于 10000，标记为'大额'"

5. 数据合并

将多个来源的数据合并成一个统一格式，保留来源标记

数据处理挑战（10 分钟）

任务：有 3 个 Excel 文件，分别包含不同地区的农产品价格

要求：合并成一个文件，按地区分类，计算平均价格，标记最高价和最低价

关键提示

先拿 1 个文件跑通，再扩展到 3 个。批量处理的本质是"重复执行同一个正确操作"。

联调关键：数据格式

[ 数据流示意图 ]

QClaw 输出 → JSON/CSV → WorkBuddy 读取 → 清洗 → 标准化 JSON → Trae 读取 → 展示

联调的关键是数据格式。QClaw 输出的文件，WorkBuddy 要能正确读取；WorkBuddy 处理后的数据，Trae 要能正确展示。

常见报错与解决

  • "批量处理 10 个文件总是报错" → 先拿 1 个文件跑通
  • "格式转换后数据乱了" → 转换前先备份，尝试 UTF-8 重新保存
  • "自动化流程保存后找不到了" → 命名时加日期和用途，如"0401_价格数据清洗"

产出模板

预期产出样例

processed-data.json：包含统一字段（id, title, category, date, source, confidence），UTF-8 编码，字段名和接口约定一致

Day 4 · 知识储备

范围控制

MVP 定义：先跑通，再完美

什么是 MVP？

Minimum Viable Product（最小可行产品）：不要一上来就想做完美的东西，先做一个"能跑通核心流程"的最简单版本。

以"农产品价格监测系统"为例

MVP 版本：QClaw 搜集 5 个网站的价格 → WorkBuddy 整理成表格 → Trae 显示出来
非 MVP 功能（后续迭代）：自动预警、历史趋势图、多地区对比

互动：MVP 边界确认（5 分钟）

每组用 1 句话描述 MVP 边界，讲师记录在白板上：

  • 对边界过大的组，现场协助裁剪
  • 对边界过小的组，鼓励增加一个展示亮点

常见疑问

"MVP 是不是太简单了？" → 评委看的是"有没有跑通核心流程"，不是功能多少。一个能稳定跑通的简单系统，比一堆半成品强 100 倍。

MVP 检查清单

检查项标准是否达标
QClaw 采集能自动采集并输出结构化文件□
WorkBuddy 处理能读取 QClaw 输出并生成报告□
Trae 展示有界面、有逻辑、有数据□
数据联调数据能从 QClaw 流入 WorkBuddy□
演示流畅10 分钟内能展示完核心功能□

常见错误预判

  • "沉迷调界面" → 下午不许碰 CSS，先确保功能完整
  • "采集结果太多处理不完" → 限制采集量：先只做 10 条
  • "组内有人闲着" → 分配：记录文档、测试边界、准备需求
  • "想换选题" → Day 4 一律不允许，强制在当前选题内裁剪

Day 4 · 排错指南

常见问题

联调常见问题排查指南

1

接口 404

现象：浏览器控制台显示 "404 Not Found"

解决：检查文件路径拼写 → 确认文件已生成 → 使用 Live Server 运行

2

CORS 报错

现象："Access-Control-Allow-Origin" 错误

解决：使用 Live Server 插件 → 或配置本地代理 → 避免直接打开文件

3

数据格式不匹配

现象：字段显示 undefined 或数据错位

解决：对比 JSON 字段名和代码中的字段名 → 统一接口契约 → 添加默认值

4

页面白屏

现象：浏览器显示空白，无任何内容

解决：F12 打开控制台查看 JS 报错 → 复制报错给 Trae 修复 → 检查数据文件路径

5

样式错乱

现象：卡片重叠、文字溢出、布局混乱

解决：检查 CSS 文件是否正确加载 → 在 Trae 中描述问题让 AI 修复 → 优先确保功能，样式最后调
