知识储备

OpenClaw 架构解析：四层设计，万能平台

1

接入层 —— 用户从哪里来

负责接收用户的请求，支持多种接入渠道。用户可以通过微信直接对话、通过网页界面操作、或者通过 API 接口将 OpenClaw 集成到自己的系统中。

例如：微信机器人、网页聊天窗口、企业系统对接

2

能力层（Skills）—— 能做什么

这是 OpenClaw 的"应用商店"。每个 Skill 是一个独立的功能模块，可以安装、卸载、自定义开发。Skills 让 Agent 从"会聊天"变成"会做事"。

例如：搜索 Skill、文件处理 Skill、微信推送 Skill、天气查询 Skill

3

核心层 —— 大脑怎么思考

LLM 推理引擎是 Agent 的"大脑"。它负责理解用户意图、拆解任务步骤、决定调用哪些 Skills、整合返回结果。支持切换不同的底层大模型。

例如：GPT-4、Claude、国产大模型均可接入

4

数据层 —— 记住什么

负责长期记忆和知识存储。包括对话历史、用户偏好、知识库文档等。让 Agent 越用越懂你，不同会话之间保持连续性。

例如：向量数据库、关系数据库、文件存储系统

"万能平台"的核心特点

  • 模块化设计：像搭积木一样组合功能
  • 多渠道接入：一个后端，多个前端
  • 可替换大脑：底层 LLM 随时切换
  • 开放生态：社区贡献数千个 Skills
  • 企业级部署：支持私有化、集群化

类比理解

OpenClaw 就像一个智能手机：接入层是触屏和按键（交互方式），能力层是安装的 App（Skills），核心层是处理器和操作系统（LLM 引擎），数据层是存储空间（记忆和文件）。
