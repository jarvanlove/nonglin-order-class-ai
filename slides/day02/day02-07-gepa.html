知识储备

GEPA：让 Agent 像生物一样进化的秘密

GEPA 是什么？

GEPA = 遗传（Genetic）+ 帕累托（Pareto）+ 提示进化（Prompt Evolution）。这是 Hermes Agent 实现自我进化的核心算法。

通俗解释

想象你让 Agent 写一份实验报告。第一次，它用一种格式；第二次，换另一种格式；第三次，再换一种……每次完成后，Agent 都会评估"哪种格式效果最好"。效果好的格式被"保留"（遗传），效果差的被"淘汰"。经过多轮迭代，Agent 自动找到最适合你的写法——就像生物进化一样，越用越懂你。

类比：达尔文雀的进化

达尔文观察到，加拉帕戈斯群岛上的雀鸟因为食物来源不同，喙的形状逐渐分化——吃坚果的喙变厚，吃昆虫的喙变细。

GEPA 的原理类似：Agent 面对不同任务（不同"食物"），自动演化出最适合的提示词策略（不同"喙形"）。不是人设计出来的，是它自己"长"出来的。

GEPA 进化循环

1. 生成（Generate）

Agent 随机生成多种提示词变体，尝试不同表达方式

↓

2. 评估（Evaluate）

用预设标准（准确率、用户满意度等）评估每种变体的效果

↓

3. 选择（Select）

保留效果好的变体，淘汰效果差的（帕累托最优筛选）

↓

4. 进化（Evolve）

对保留的变体进行交叉、变异，产生下一代提示词

↓

循环往复...

每一轮都比上一轮更好，Agent 持续自我优化

为什么这很重要？

传统 Agent 的能力上限取决于开发者的设计；GEPA Agent 的能力上限取决于使用频率——用得越多，越聪明。这意味着你的 Agent 是独一无二的，因为它在你的使用场景中进化出了专属于你的能力。
