## 模块一：OpenClaw 架构解析（50 分钟）

### 1.1 项目背景与定位

**讲师话术**：

> **"OpenClaw 是目前 GitHub 上 Star 数最多的开源 AI Agent 项目，超过 33 万。创始人 Peter Steinberger 是奥地利开发者，今年 2 月加入了 OpenAI，项目已经转交给开源基金会运营。这个数字什么概念？它比 Linux 内核早期的增长还快。"**

OpenClaw 的核心定位：**开源 AI Agent 平台**，不是单一工具，而是一个可以让开发者和普通用户都能搭建自己 Agent 的"操作系统"。

### 1.2 系统架构图描述

OpenClaw 采用**分层架构**，从上到下可分为四层：

**第一层：交互层（Interface Layer）**
- 支持 20+ 消息渠道接入
- 包括：Web 界面、Slack、Discord、Telegram、微信、邮件、短信等
- 核心设计思想：用户在哪，Agent 就在哪

**第二层：核心引擎层（Core Engine）**
- **对话管理器**：维护多轮对话上下文，处理用户意图识别
- **任务调度器**：将复杂任务拆解为子任务，分配给不同 Skill 执行
- **记忆管理器**：短期对话记忆 + 长期用户偏好记忆
- **模型路由层**：根据任务类型自动选择最优模型（GPT-4、Claude、本地模型等）

**第三层：Skills 生态层（Skills Layer）**
- Skill = Agent 的"技能插件"
- 每个 Skill 是一个独立功能模块（如：查天气、发邮件、操作 Excel、调用 API）
- ClawHub 市场已有 **13,700+** 个 Skills
- Skill 开发门槛：会写 Python 就能开发，社区贡献活跃

**第四层：基础设施层（Infrastructure Layer）**
- 支持本地部署、云端部署、混合部署
- 数据库：PostgreSQL / SQLite 可选
- 向量数据库：用于长期记忆检索
- 沙箱执行环境：确保 Skill 代码安全运行

**讲师话术**：

> **"想象 OpenClaw 是一个餐厅。交互层是门口的各种点餐渠道——你可以在微信下单、在 Slack 下单、或者直接走进来。核心引擎是厨房调度系统，决定哪个菜给哪个厨师做。Skills 是厨师们的手艺，有人擅长做川菜，有人擅长做粤菜。基础设施就是厨房的灶台、冰箱、消毒柜。"**

### 1.3 多渠道接入说明

OpenClaw 的多渠道能力不是简单的"多开几个聊天窗口"，而是**统一的身份识别和状态同步**：

- 你在微信上让 OpenClaw 查了一个资料，回到 Web 界面继续对话，它能记住刚才的上下文
- 在 Slack 里布置的任务，可以通过邮件收到进度通知
- 每个渠道的用户身份统一映射到同一个 Agent 实例

**关键设计：渠道适配器（Channel Adapter）**
- 每个渠道有一个适配器，负责协议转换
- 微信适配器处理微信消息格式 → 转换为 OpenClaw 内部标准格式
- Slack 适配器处理 Slack 的 Block Kit 格式 → 同样转换为标准格式
- 新增一个渠道，只需开发一个适配器，无需改动核心逻辑

### 1.4 Skills 生态与 ClawHub 市场

**ClawHub 数据（截至 2026 年 4 月）**：
- 总 Skills 数量：**13,700+**
- 月新增 Skills：约 400-600 个
- 下载量 Top 3 类别：办公自动化（23%）、数据分析（18%）、开发工具（15%）
- 官方认证 Skills：约 1,200 个（带蓝标，经过安全审核）

**Skill 的典型结构**：
```
skill-name/
  ├── manifest.json      # Skill 元信息（名称、版本、作者、权限）
  ├── main.py           # 主逻辑入口
  ├── config.yaml       # 配置项（用户可自定义）
  └── README.md         # 使用说明
```

**讲师话术**：

> **"13,700 个 Skills 是什么概念？相当于有 13,700 个 mini App 在为你工作。你想让 Agent 帮你操作 Excel，不需要自己写代码，去 ClawHub 搜 'excel'，安装一个 Skill 就行。这就是开源生态的力量——一个人做不完的事，一万个人一起做。"**

### 1.5 33 万 Stars 的增长逻辑

OpenClaw 的快速增长有三个核心驱动力：

1. **时机精准**：2023 年初发布，正好赶上 GPT-4 发布后的 Agent 需求爆发期
2. **架构开放**：不绑定任何模型，用户可以用 OpenAI、Anthropic、本地模型，甚至国产模型
3. **社区运营**：创始人 Peter Steinberger 个人技术影响力 + 积极的贡献者激励计划

## 模块二：OpenClaw vs ChatGPT 核心区别对照（15 分钟）

**讲师话术**：

> **"很多同学问我：OpenClaw 和 ChatGPT 有什么区别？我直接用 ChatGPT 不行吗？下面的对照表会告诉你，它们不是替代关系，而是不同维度的工具。"**

| 对比维度 | OpenClaw | ChatGPT |
|---------|---------|---------|
| **交互模式** | 多轮任务驱动，可主动调用工具执行操作 | 对话问答，被动响应用户输入 |
| **运行环境** | 可本地部署，数据不出内网 | 云端服务，数据上传至 OpenAI 服务器 |
| **可扩展性** | 13,700+ Skills 插件，可自定义开发 | 固定功能集，仅通过 GPTs 有限扩展 |
| **数据控制** | 完全自主，数据库、模型、代码都在本地 | 受 OpenAI 隐私政策约束 |
| **模型选择** | 支持 20+ 种模型自由切换 | 仅 OpenAI 自有模型 |
| **开源性** | 完全开源（MIT 协议），可二次开发 | 闭源商业产品 |
| **多渠道路由** | 支持 20+ 渠道统一接入 | 仅官方 App / Web / API |
| **记忆能力** | 持久化记忆，跨会话保持 | 单会话记忆，或有限的长期记忆 |
| **适用场景** | 企业自动化、重复性工作流、私有化部署 | 通用问答、创意写作、快速咨询 |

**核心结论**：
- ChatGPT 是"超级大脑"，擅长思考和生成内容
- OpenClaw 是"数字员工"，擅长执行重复性任务和集成多系统

**讲师话术**：

> **"简单说：ChatGPT 是你脑子里的顾问，你问它答；OpenClaw 是你办公桌上的实习生，你告诉它目标，它自己去查资料、填表格、发邮件。两者结合才是完整的工作流。"**

## 模块三：Hermes Agent 深度解析（60 分钟）

### 3.1 Nous Research 背景

**讲师话术**：

> **"如果说 OpenClaw 是 Agent 世界的'安卓系统'，那么 Hermes Agent 就是 Agent 世界的'特斯拉'——它不追求功能最多，而是在一个点上做到极端：自我进化。"**

**Nous Research** 是一家专注于开源 AI 研究的机构，核心理念：
- 不做封闭产品，只做开源研究
- 相信 AI 应该具备"自我改进"能力，而不是依赖人类不断调优
- 团队规模不大（约 30 人），但产出极具影响力

**Hermes Agent 关键数据**：
- GitHub Stars：**103,000+**
- 开源协议：MIT（完全自由使用）
- 发布时间：2026 年 2 月 25 日（非常新的项目）
- 支持模型：**400+** 种（包括国产模型如 DeepSeek、ChatGLM）

### 3.2 GEPA：遗传-帕累托提示进化机制

这是 Hermes Agent 最核心的技术创新，也是最难理解的部分。我们用通俗方式拆解。

**问题背景**：
传统 AI Agent 的提示词（Prompt）由人类工程师编写，一旦写好就固定了。但不同任务需要不同的提示策略，人类手动调优效率极低。

**GEPA 的核心思想**：
让 Agent **自己进化自己的提示词**，就像生物进化一样——优胜劣汰，适者生存。

**GEPA 三步循环**：

**Step 1：生成（Genetic）**
- Agent 根据当前任务，生成多个不同版本的提示词变体
- 每个变体像一条"DNA"，包含不同的表达策略、思考步骤、输出格式

**Step 2：评估（Pareto）**
- 每个提示词变体被用于执行同一批测试任务
- 评估指标：准确率、执行速度、资源消耗、输出稳定性
- 使用"帕累托最优"原则筛选：不存在另一个变体在所有指标上都比它好

**Step 3：进化（Evolution）**
- 优秀的提示词变体被保留，并进行"交叉"和"变异"
- 交叉：两个优秀变体各取一部分，组合成新变体
- 变异：随机修改某些词语或结构，探索新的可能性
- 循环回到 Step 1，持续迭代

**讲师话术**：

> **"想象你是一个厨师，要做一道菜。传统方式是：你写个菜谱，以后每次都按这个做。GEPA 方式是：你同时做 10 个版本，让客人打分，把得分高的做法保留下来，再混搭、改良，做出第 11、12 个版本。周而复始，你的菜谱会越来越好吃——而且不需要你亲自试每一个版本，AI 自己就在后台跑这个进化过程。"**

**GEPA 的实际效果**：
- 在代码生成任务上，经过 50 轮进化后的提示词，比人工编写的基准提示词准确率提升 23%
- 在数据分析任务上，进化后的提示词减少了 40% 的冗余步骤
- 进化过程完全自动化，人类只需设定评估标准

### 3.3 持久记忆三层架构

Hermes Agent 的记忆系统比 OpenClaw 更复杂，分为三层：

**第一层：工作记忆（Working Memory）**
- 作用：当前对话的短期上下文
- 容量：最近 10-20 轮对话
- 特点：读写速度极快，断电即失
- 类比：人脑的前额叶，正在处理的事情

**第二层：情景记忆（Episodic Memory）**
- 作用：存储过去的任务执行记录和结果
- 实现方式：向量数据库 + 时间戳索引
- 检索方式：语义相似度搜索（"我之前做过类似的事吗？"）
- 类比：人脑的海马体，记得"上周三我用 Excel 处理过销售数据"

**第三层：程序记忆（Procedural Memory）**
- 作用：存储经过验证的"最佳实践"和"技能模板"
- 来源：GEPA 进化出的优秀提示词 + 成功任务的经验萃取
- 特点：高度抽象，跨任务复用
- 类比：人脑的肌肉记忆，骑自行车一旦学会就不会忘

**讲师话术**：

> **"这三层记忆的关系，就像你学开车。工作记忆是你现在握着方向盘、看着路况；情景记忆是你记得'上个月我在雨天开过一次高速，要减速'；程序记忆是你不用想就知道'踩离合、挂挡、松手刹'。Hermes Agent 的设计就是让 AI 也能积累这种从经验中提炼出来的'肌肉记忆'。"**

### 3.4 与 OpenClaw 的差异化定位

| 维度 | OpenClaw | Hermes Agent |
|------|---------|-------------|
| **核心哲学** | 生态优先：连接一切，集成一切 | 进化优先：自我改进，越用越聪明 |
| **架构重点** | 多渠道接入 + Skills 插件市场 | 提示词进化 + 持久记忆 |
| **扩展方式** | 人类开发 Skills 扩展功能 | AI 自我进化提示词提升能力 |
| **适用任务** | 重复性工作流、多系统集成 | 复杂推理、需要经验积累的任务 |
| **部署复杂度** | 中等（需要配置数据库、渠道适配器） | 较低（单体式部署，内置向量库） |
| **社区规模** | 极大（33 万 Stars，生态成熟） | 快速增长（10 万 Stars，较新） |
| **模型支持** | 20+ 种 | 400+ 种（包括大量国产模型） |

**讲师话术**：

> **"选型建议：如果你需要让 AI 帮你对接微信、Slack、Excel、邮件，做一套自动化办公流——选 OpenClaw。如果你需要让 AI 解决一个复杂的、需要反复试错的问题，而且希望它越用越顺手——选 Hermes Agent。当然，两者也可以结合：用 OpenClaw 做调度，把复杂任务丢给 Hermes Agent 处理。"**

## 附录：学生常见问题预设

**Q1：OpenClaw 和 Hermes Agent 国内能用吗？**
A：理论课可以介绍，但实践需要网络环境支持。OpenClaw 可以本地部署并使用国产模型；Hermes Agent 同样支持本地部署 + 国产模型。课程中实践环节使用 WorkBuddy 正是因为网络限制。

**Q2：我不是计算机专业的，这些架构图我听不懂怎么办？**
A：记住三个比喻就行——餐厅（OpenClaw）、进化（GEPA）、学开车（三层记忆）。工作中你不需要画架构图，只需要知道"这个工具能做什么、不能做什么"。

**Q3：这些开源项目会不会过两年就没人维护了？**
A：OpenClaw 已经转给开源基金会，且社区极大，风险很低。Hermes Agent 较新，但 Nous Research 有持续的研究产出。更重要的是：你今天学到的"Agent 思维"是通用的，即使具体工具换了，认知不会过时。
