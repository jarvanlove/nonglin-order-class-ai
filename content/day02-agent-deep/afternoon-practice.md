## 第一阶段：WorkBuddy 安装与界面导览（30 分钟）

### 1.1 安装步骤

#### Windows 用户

**步骤 1：下载安装包**
1. 打开浏览器，访问官网：`https://workbuddycn.com`
2. 点击页面右上角 **"下载"** 按钮（绿色按钮，带 Windows 图标）
3. 选择 **"Windows 版"** → 点击 **"立即下载"**
4. 等待 `WorkBuddySetup.exe` 下载完成（约 180MB）

**步骤 2：运行安装程序**
1. 打开下载文件夹，双击 `WorkBuddySetup.exe`
2. 若弹出 **"Windows 已保护你的电脑"** 提示（SmartScreen）：
   - 点击 **"更多信息"**
   - 点击 **"仍要运行"**
3. 安装向导启动后，点击 **"下一步"**
4. 选择安装路径（建议保持默认 `C:\Program Files\WorkBuddy`），点击 **"下一步"**
5. 勾选 **"创建桌面快捷方式"**，点击 **"安装"**
6. 等待安装完成（约 1-2 分钟），点击 **"完成"**

**步骤 3：首次启动**
1. 双击桌面 **WorkBuddy** 图标启动
2. 首次启动会弹出 **登录窗口**：
   - 点击 **"微信扫码登录"**
   - 使用手机微信扫描屏幕上的二维码
   - 手机上点击 **"确认登录"**
3. 登录成功后进入主界面

#### macOS 用户

**步骤 1：下载安装包**
1. 打开浏览器，访问官网：`https://workbuddycn.com`
2. 点击页面右上角 **"下载"** 按钮
3. 选择 **"Mac 版"** → 点击 **"立即下载"**
4. 等待 `WorkBuddy.dmg` 下载完成（约 200MB）

**步骤 2：安装应用**
1. 打开下载文件夹，双击 `WorkBuddy.dmg`
2. 将 **WorkBuddy 图标** 拖拽到 **Applications 文件夹** 图标上
3. 等待复制完成（约 30 秒）
4. 双击 Applications 中的 WorkBuddy 启动

**步骤 3：首次启动（macOS 安全设置）**
1. 若弹出 **"无法打开 WorkBuddy"**（因为无法验证开发者）：
   - 点击 **"取消"**
   - 打开 **系统设置 → 隐私与安全性**
   - 滚动到 **"安全性"** 区域
   - 点击 **"仍要打开"**（在 "WorkBuddy 已被阻止使用" 旁边）
   - 再次点击 **"打开"**
2. 首次启动会弹出登录窗口，按 Windows 步骤 3 完成微信扫码登录

#### Windows / Mac 差异速查表

| 操作 | Windows | macOS |
|------|---------|-------|
| 安装包格式 | `.exe` | `.dmg` |
| 安装方式 | 向导式安装 | 拖拽到 Applications |
| 安全拦截 | SmartScreen → "更多信息" → "仍要运行" | 系统设置 → 隐私与安全性 → "仍要打开" |
| 默认安装路径 | `C:\Program Files\WorkBuddy` | `/Applications/WorkBuddy.app` |
| 卸载方式 | 控制面板 → 卸载程序 | 直接拖入废纸篓 |
| 快捷键差异 | Ctrl + C/V | Command + C/V |

**区域 1：顶部导航栏**

### 1.3 模型切换

2. 弹出下拉菜单，显示可用模型列表：

### 1.4 权限模式说明

WorkBuddy 有三种权限模式，决定 AI 能执行的操作范围：

**Ask 模式（询问模式）**
- 图标：问号 `?`
- 权限：AI 只能回答问题，**不能操作你的电脑**
- 适用：咨询、问答、文本生成
- 安全级别：最高（完全只读）

**Plan 模式（规划模式）**
- 图标：清单 `☰`
- 权限：AI 可以生成操作计划，但**执行前会征求你的确认**
- 适用：需要多步骤的复杂任务
- 安全级别：中等（每一步都需确认）

**Craft 模式（制作模式）**
- 图标：锤子 `🔨`
- 权限：AI 可以直接执行文件操作（读、写、重命名、移动）
- 适用：批量处理、自动化任务
- 安全级别：较低（需谨慎使用，确认操作范围）

**切换方式**：
1. 在对话区顶部，点击当前模式图标
2. 弹出三个模式选项
3. 点击目标模式完成切换

**重要提示**：
- 下午的所有练习都需要在 **Craft 模式** 下进行
- 首次切换到 Craft 模式时，会弹出权限说明弹窗，勾选 **"我已了解风险"** 后点击 **"确认开启"**

## 第二阶段：核心功能分步练习（60 分钟）

### 练习 1：本地文件处理（20 分钟）

**目标**：让 WorkBuddy 读取本地文件夹，列出文件清单并提取关键信息。

**步骤**：

1. **切换至 Craft 模式**
   - 点击对话区顶部的模式图标
   - 选择 **Craft 模式**
   - 勾选风险提示，点击 **确认开启**

2. **上传文件夹**
   - 点击底部输入区的 **回形针图标**
   - 选择 **"上传文件夹"**
   - 在弹出的文件浏览器中，导航到 `day02-practice-data/raw_files/`
   - 点击 **"选择文件夹"**
   - 等待上传完成（界面显示 "已加载文件夹"）

3. **发送指令**
   - 在输入框中输入：
     ```
     请列出这个文件夹里的所有文件，按文件类型分类，并统计每种类型的数量。
     ```
   - 按 **Enter** 或点击 **发送按钮**

4. **查看结果**
   - WorkBuddy 会返回一个分类列表，例如：
     ```
     图片文件 (3): img_001.jpg, img_002.jpg, img_003.jpg
     文本文件 (2): doc_old_1.txt, doc_old_2.txt
     ```

5. **进阶指令**
   - 继续输入：
     ```
     请提取所有图片文件的文件名（不含扩展名），生成一个清单。
     ```
   - WorkBuddy 会输出纯文件名列表

**常见问题排查**：

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 上传文件夹按钮灰色不可用 | 未开启 Craft 模式 | 先切换至 Craft 模式 |
| 提示"无法访问该路径" | 文件夹路径含中文特殊字符或权限不足 | 将文件夹复制到桌面再上传 |
| 列出的文件不完整 | 文件夹内文件过多（超过 100 个）或含隐藏文件 | 分批上传，或在指令中指定"包括隐藏文件" |
| 文件类型识别错误 | 文件扩展名被修改过 | 在指令中补充"根据文件头内容判断真实类型" |
| 上传后无响应超过 30 秒 | 网络延迟或文件过大 | 取消后重新上传，或压缩文件夹 |

### 练习 2：Excel 数据清洗（20 分钟）

**目标**：使用 WorkBuddy 清洗一份存在空值、重复、格式混乱的销售数据。

**数据说明**：
`sales_data.xlsx` 包含以下问题：
- 第 3 行、第 7 行存在空值
- 第 5 行和第 12 行数据重复
- "日期"列格式不统一（有的是 2024/1/15，有的是 2024-01-15）
- "金额"列有的带人民币符号，有的是纯数字

**步骤**：

1. **上传 Excel 文件**
   - 点击底部输入区的 **回形针图标**
   - 选择 **"上传文件"**
   - 选择 `day02-practice-data/sales_data.xlsx`
   - 等待上传完成

2. **查看数据概况**
   - 输入指令：
     ```
     请读取这个 Excel 文件，告诉我它有多少行、多少列，列名分别是什么。
     ```
   - WorkBuddy 会返回数据的基本信息

3. **数据清洗**
   - 输入指令：
     ```
     请帮我清洗这份数据：
     1. 删除完全重复的行
     2. 填充空值：数值列用0填充，文本列用"未填写"填充
     3. 统一日期格式为 YYYY-MM-DD
     4. 金额列统一为纯数字，去掉人民币符号和千分位逗号
     5. 将清洗后的数据保存到 output/cleaned_sales.xlsx
     ```
   - WorkBuddy 会显示执行计划，点击 **"确认执行"**
   - 等待处理完成（约 10-20 秒）

4. **验证结果**
   - 打开 `day02-practice-data/output/cleaned_sales.xlsx`
   - 检查：
     - 重复行是否已删除
     - 空值是否已填充
     - 日期格式是否统一
     - 金额是否为纯数字

5. **生成数据摘要**
   - 输入指令：
     ```
     基于清洗后的数据，生成一份摘要：总销售额、平均订单金额、订单数量。
     ```
   - WorkBuddy 会返回计算结果

**常见问题排查**：

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 提示"无法解析 Excel 文件" | 文件是 .xls 老格式或已损坏 | 用 Excel 打开后另存为 .xlsx 格式再上传 |
| 清洗后数据行数没变 | WorkBuddy 未识别出重复行 | 在指令中明确指定"根据 A 列和 B 列联合判断是否重复" |
| 日期格式未统一 | 原数据中存在非标准日期文本 | 在指令中补充"无法识别的日期标记为'格式错误'" |
| 金额列仍含符号 | 符号为全角字符（如￥） | 指令中写"去掉所有非数字字符，保留小数点" |
| 保存失败提示"路径不存在" | output 文件夹未创建 | 先在文件管理器中手动创建 output 文件夹 |
| 中文列名识别错误 | 编码问题 | 确保 Excel 保存为 UTF-8 编码，或在指令中逐列说明 |

### 练习 3：批量重命名与归档（20 分钟）

**目标**：使用 WorkBuddy 对 `raw_files` 中的文件进行批量重命名，并按类型归档到不同文件夹。

**步骤**：

1. **确认 Craft 模式已开启**
   - 检查对话区顶部显示 **Craft 模式**

2. **上传文件夹**
   - 上传 `day02-practice-data/raw_files/`（同练习 1）

3. **批量重命名**
   - 输入指令：
     ```
     请将这个文件夹中的所有图片文件重命名为 "photo_001.jpg", "photo_002.jpg" 的格式，
     文本文件重命名为 "note_001.txt", "note_002.txt" 的格式。
     保持原有扩展名不变，编号从 001 开始按字母顺序排列。
     ```
   - WorkBuddy 会列出将要执行的重命名操作预览
   - 检查预览列表无误后，点击 **"确认执行"**

4. **按类型归档**
   - 输入指令：
     ```
     请在这个文件夹下创建两个子文件夹："images" 和 "documents"，
     然后将所有图片文件移动到 "images"，所有文本文件移动到 "documents"。
     ```
   - 确认预览后点击 **"确认执行"**

5. **验证结果**
   - 打开文件管理器，查看 `raw_files` 文件夹
   - 应看到：
     ```
     raw_files/
     ├── images/
     │   ├── photo_001.jpg
     │   ├── photo_002.jpg
     │   └── photo_003.jpg
     └── documents/
         ├── note_001.txt
         └── note_002.txt
     ```

**常见问题排查**：

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 重命名后文件丢失 | 文件名冲突导致覆盖 | 在指令中补充"如遇冲突，在文件名后加 _1" |
| 文件夹创建失败 | 权限不足 | 右键 WorkBuddy 图标 → "以管理员身份运行"（Windows） |
| 移动后原文件夹还有文件 | 某些文件类型未被识别 | 在指令中明确列出所有要处理的扩展名 |
| 编号顺序混乱 | 文件系统排序与预期不同 | 指令中补充"按文件修改时间从早到晚编号" |
| 提示"操作被拒绝" | 文件被其他程序占用 | 关闭可能占用文件的程序（如 Excel、图片查看器） |
| 归档后找不到文件 | 路径理解错误 | 在指令中明确"在原始文件夹内创建子文件夹" |

## 第三阶段：综合任务——周报自动化生成（105 分钟）

### 任务目标

使用 WorkBuddy 完成一份完整的周报生成流程：
1. 从多个源文件（Excel 销售数据、Word 会议纪要、项目进度表）提取关键信息
2. 生成数据可视化图表描述
3. 输出一份格式化的周报 Word 文档

### 源文件说明

| 文件 | 内容 | 需要提取的信息 |
|------|------|--------------|
| `sales_data.xlsx` | 本周销售数据 | 总销售额、Top 3 产品、环比变化 |
| `meeting_notes_1.docx` | 周一例会纪要 | 会议决议、待办事项、负责人 |
| `meeting_notes_2.docx` | 周三项目评审会 | 项目风险、里程碑变更 |
| `project_status.xlsx` | 项目进度跟踪 | 各项目完成百分比、阻塞问题 |

### 步骤详解

#### Step 1：数据提取（30 分钟）

1. **上传所有源文件**
   - 点击 **回形针图标** → **"上传文件"**
   - 按住 **Ctrl**（Windows）或 **Command**（Mac）多选以下文件：
     - `sales_data.xlsx`
     - `meeting_notes_1.docx`
     - `meeting_notes_2.docx`
     - `project_status.xlsx`
   - 点击 **"打开"** 上传

2. **提取销售数据摘要**
   - 输入指令：
     ```
     请读取 sales_data.xlsx，提取以下信息并用 bullet points 列出：
     - 本周总销售额
     - 销售额最高的三个产品及其金额
     - 与上周相比的增长率（假设上周总销售额为 45000）
     ```
   - 记录 WorkBuddy 返回的结果（后面会用到）

3. **提取会议纪要要点**
   - 输入指令：
     ```
     请读取两个 Word 文件，分别提取：
     1. 会议决议（Decisions）
     2. 待办事项（Action Items），标注负责人和截止日期
     3. 识别的风险或问题
     ```
   - WorkBuddy 会分别列出两个会议的要点

4. **提取项目进度**
   - 输入指令：
     ```
     请读取 project_status.xlsx，列出：
     - 各项目名称及完成百分比
     - 状态为"阻塞"的项目及阻塞原因
     - 预计本周内完成的项目
     ```
   - 记录返回结果

#### Step 2：生成图表数据（20 分钟）

WorkBuddy 本身不直接生成图片格式的图表，但它可以生成**可用于插入图表的结构化数据**和**图表描述**。

1. **生成销售图表数据**
   - 输入指令：
     ```
     基于 sales_data.xlsx 的数据，生成一个适合制作柱状图的表格数据：
     列：产品名称 | 本周销售额 | 上周销售额
     按销售额从高到低排列，输出为 Markdown 表格格式。
     ```
   - WorkBuddy 会输出 Markdown 表格

2. **生成项目进度图表数据**
   - 输入指令：
     ```
     基于 project_status.xlsx，生成适合制作甘特图或进度条图的数据：
     列：项目名称 | 计划完成日期 | 实际完成百分比 | 状态
     输出为 Markdown 表格。
     ```
   - 记录输出结果

3. **生成图表插入说明**
   - 输入指令：
     ```
     请为上述两组数据分别写一段"图表插入说明"，告诉我：
     1. 推荐用什么类型的图表展示
     2. 图表标题建议
     3. 关键数据点标注建议
     ```
   - 这些说明将用于手动在 Word 中插入图表（或使用 WorkBuddy 的高级功能自动生成）

#### Step 3：生成格式化周报（40 分钟）

1. **生成周报正文**
   - 输入指令：
     ```
     请基于我们刚才提取的所有信息，生成一份周报正文，要求：
     - 标题：第 X 周工作周报（2026年4月第4周）
     - 结构：
       一、本周工作概述（2-3句话总结）
       二、销售数据（含关键数字）
       三、会议决议与待办
       四、项目进度
       五、下周计划（基于待办事项和项目进度合理推测）
     - 语言风格：正式、简洁、数据驱动
     ```
   - WorkBuddy 会生成完整的周报文本

2. **保存为 Word 文档**
   - 输入指令：
     ```
     请将上述周报内容保存为 Word 文档，路径：day02-practice-data/output/weekly_report.docx
     格式要求：
     - 标题用"标题 1"样式
     - 各章节用"标题 2"样式
     - 正文用"正文"样式
     - 关键数据加粗
     ```
   - 确认执行后，WorkBuddy 会生成 `.docx` 文件

3. **验证文档**
   - 打开 `output/weekly_report.docx`
   - 检查：
     - 标题层级是否正确
     - 关键数据是否加粗
     - 内容是否完整
     - 有无明显格式错误

4. **手动插入图表（可选进阶）**
   - 如果 WorkBuddy 支持直接生成图表插入文档，按提示操作
   - 否则，手动操作：
     1. 在 Word 中定位到"销售数据"章节
     2. 点击 **插入 → 图表 → 柱状图**
     3. 将 Step 2 生成的 Markdown 表格数据粘贴到 Excel 数据源中
     4. 关闭数据源，图表自动更新
     5. 调整图表标题为 WorkBuddy 建议的标题

#### Step 4：优化与迭代（15 分钟）

1. **检查完整性**
   - 输入指令：
     ```
     请检查这份周报是否遗漏了任何重要信息，对比原始四个文件的内容。
     ```
   - 根据 WorkBuddy 的提示补充遗漏内容

2. **语言润色**
   - 输入指令：
     ```
     请将周报中"待办事项"部分的表述改为更正式的商务用语，
     将"问题"改为"待协调事项"或"风险点"。
     ```

3. **最终保存**
   - 确认无误后，保存最终版本
   - 文件路径：`day02-practice-data/output/weekly_report_final.docx`

### 综合任务常见问题排查

| 问题现象 | 可能原因 | 解决方法 |
|---------|---------|---------|
| 上传多个文件后 AI 只识别了其中一个 | 文件上传有延迟，AI 在全部上传完成前已处理 | 等待所有文件显示"已上传"后再发送指令 |
| 提取的信息与文件内容不符 | AI 混淆了多个文件的内容 | 在指令中明确指定文件名，如"读取 sales_data.xlsx" |
| 生成的周报内容空洞 | 指令不够具体 | 在指令中明确要求"包含具体数字"、"引用原文" |
| Word 文档保存后打不开 | 格式指令过于复杂导致生成错误 | 简化格式要求，先保存纯文本再手动排版 |
| 图表数据与文字数据不一致 | 两次读取文件的时间差导致 | 在同一轮对话中完成数据提取和图表生成 |
| 周报中出现了假设性内容 | AI 为补全内容进行了"幻觉"生成 | 在指令中强调"只基于提供的文件内容，不要编造" |
| 文件路径错误导致保存失败 | 使用了相对路径或路径含特殊字符 | 使用绝对路径，或确保 output 文件夹已存在 |
