| **定位** | 国外 AI IDE 标杆，国外开发者首选 |

#### 4.1.2 模型配置（关键步骤）

**【讲师话术】**

### 4.2 Trae 界面导览

**【需要配图：Trae 界面截图，标注各区域】**

界面四大区域：

1. **左侧边栏**：文件资源管理器（跟 VS Code 一样）
2. **中间编辑区**：代码编辑主窗口
3. **右侧/底部 AI 面板**：与 AI 对话的区域（Trae 的核心差异点）
4. **顶部工具栏**：运行、调试、Git 等快捷按钮

**【讲师话术】**
> "Trae 的界面跟 VS Code 非常像，因为底层就是基于 VS Code 的。如果你之前用过 VS Code，基本零学习成本。没用过也没关系，今天只用到最基础的功能。"

### 4.3 三种模式详解

**【讲师话术】**
> "Trae 有三种模式，这是它最核心的设计。理解这三种模式的区别，你就掌握了 Trae 的精髓。"

#### 4.3.1 Builder 模式：从零构建

**定位**："你描述需求，AI 生成完整项目"

**适用场景**：
- 从零开始做一个新功能
- 编程新手不知道从何下手
- 快速搭建原型验证想法

**操作方式**：
1. 点击 Builder 按钮（界面左上角）
2. 用自然语言描述你想做什么
3. AI 会生成文件结构、代码、配置
4. 你审阅并确认，AI 自动写入文件

**示例对话**：
```
用户：帮我做一个个人介绍网页，要有姓名、照片位置、简介、联系方式四个部分，
      风格要简洁现代，用浅灰色背景。

AI：我来为你创建一个个人介绍网页。计划如下：
    1. 创建 index.html（主页面）
    2. 创建 style.css（样式文件）
    3. 使用 Flexbox 布局确保响应式

    请确认后我将开始生成代码。
```

**【讲师话术】**
> "Builder 模式是今天你们要重点掌握的。它最像'雇了一个会编程的助理'——你说需求，它执行。对于编程基础弱的同学，这是最友好的入门方式。"

#### 4.3.2 Chat 模式：对话辅助

**定位**："边写边问，AI 是你的实时顾问"

**适用场景**：
- 你已经有一些代码，需要解释或修改
- 遇到报错，不知道如何解决
- 想优化某段代码的性能或可读性

**操作方式**：
1. 选中代码或打开文件
2. 在 Chat 面板输入问题
3. AI 基于当前文件上下文回答

**示例对话**：
```
用户：这段代码是什么意思？
AI：这段代码定义了一个函数，作用是...

用户：能不能把它改得更简洁？
AI：可以，建议改为...
```

**【讲师话术】**
> "Chat 模式就像你旁边坐了一个资深工程师，你随时能问他问题。明天和后天做项目时，你们会大量用到这个模式。"

#### 4.3.3 Solo 模式：AI 主导全流程

**定位**："你审批，AI 执行全部开发流程"

**适用场景**：
- 复杂功能开发，需要多文件协作
- 你已经明确知道要做什么，不想一步步指导 AI
- 自动化批量任务

**操作方式**：
1. 描述高层次需求
2. AI 自主规划步骤、修改文件、运行测试
3. 每一步完成后向你汇报，你确认或否决
4. 循环直到完成

**【讲师话术】**
> "Solo 模式是最高级的用法，有点像 Windsurf 的 Cascade。AI 不只是帮你写代码，而是像一个自主的开发者。但这个模式对新手来说有点'黑盒'，建议你们先熟练掌握 Builder 和 Chat，再尝试 Solo。"

### 4.4 三种模式对比

| 模式 | 你的参与度 | AI 的参与度 | 适合谁 | 今天用不用 |
|------|-----------|------------|--------|-----------|
| **Builder** | 描述需求 + 审阅结果 | 生成完整代码 | 新手、快速原型 | **用** |
| **Chat** | 主动提问 + 选择性采纳 | 回答 + 建议 | 所有人 | 明天用 |
| **Solo** | 高层次指令 + 审批 | 自主规划执行 | 有经验者 | 后天尝试 |

## 五、Trae Builder 模式实战（11:00 - 12:00，60 分钟）

### 5.1 实战任务说明

**【讲师话术】**
> "现在我们来实战。今天的任务很简单：用 Trae 的 Builder 模式，做一个'个人信息展示页'。"
>
> "这个任务所有人都能完成，不管你是不是计算机专业的。因为你不写代码，你只是告诉 AI 你要什么。"

**任务要求**：
- 一个单页面网页（HTML 文件）
- 包含：姓名、专业、个人简介、一张图片位置、联系方式
- 风格：简洁、现代、浅色背景
- 技术：HTML + CSS（AI 会自动生成）

### 5.2 分步操作指南

#### 步骤 1：打开 Builder 模式

1. 打开 Trae
2. 点击界面左上角的 **"Builder"** 按钮
3. 确保模型选择的是**豆包**或 **DeepSeek**

#### 步骤 2：输入需求

在 Builder 输入框中输入以下提示词（可以直接复制）：

```
请帮我创建一个个人介绍网页，要求如下：
1. 页面标题为"我的个人主页"
2. 包含一个头像占位区域（圆形，使用占位图）
3. 显示姓名（用占位符"[你的姓名]"）
4. 显示专业/班级信息（占位符"[你的专业]"）
5. 一段个人简介（占位符"[你的简介]"，限制 100 字以内）
6. 联系方式区域：邮箱、电话（都用占位符）
7. 整体风格：浅灰色背景，白色内容卡片，圆角设计，简洁现代
8. 页面要居中显示，在手机上也能正常看（响应式）
9. 请生成一个完整的 HTML 文件，包含内联 CSS
```

**【讲师话术】**
> "提示词的质量直接决定 AI 生成结果的质量。我给大家的提示词包含了 9 个具体要求——越具体，AI 做得越好。"
>
> "注意第 8 点'响应式'，意思是网页在手机和电脑上都能正常显示。这是现代网页的基本要求。"

#### 步骤 3：审阅 AI 的 plan

AI 会先生成一个执行计划，类似：

```
我将为你创建以下文件：
1. index.html - 主页面，包含所有 HTML 结构和内联 CSS

计划使用的技术：
- HTML5 语义化标签
- CSS Flexbox 居中布局
- @media 查询实现响应式
- 圆角、阴影等现代 CSS 特性

是否确认执行？
```

点击**"确认"**或输入"确认执行"。

#### 步骤 4：查看生成的代码

AI 生成代码后，左侧文件树会出现 `index.html`。点击即可查看代码。

**方法二：Live Server（如果已安装）**

#### 步骤 6：修改占位符内容

**【讲师话术】**

需要替换的内容：

## 七、常见问答（讲师备用）

**Q：Trae 和 Cursor 到底差多少？**
> A：功能上 Trae 大约相当于 Cursor 6-9 个月前的水平。核心差异在：生态成熟度（插件少）、多文件协作能力（弱一些）、上下文理解深度。但对于学习和基础开发，Trae 完全够用。

**Q：我是非计算机专业的，真的不用写代码吗？**
> A：今天的任务你确实不需要手写代码。但明天开始的项目中，你可能需要理解 AI 生成的代码、做少量修改。这就像你不需要会造车，但需要会开车、会看仪表盘。

**Q：Trae 个人版免费，那它怎么赚钱？**
> A：目前 Trae 靠企业版收费（69 元/月/席位）和字节的品牌效应。对学生来说，免费版功能完全足够。

**Q：我电脑配置低，Trae 能跑吗？**
> A：Trae 对配置要求不高，8GB 内存就能流畅运行。如果实在太卡，可以关闭一些不用的功能（如 Git 集成）。
