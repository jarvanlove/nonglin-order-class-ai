## 第 4 页：国外 AI 编程工具概览

- 全球开发者首选：Cursor / Windsurf / GitHub Copilot / Claude Code / Zed AI
- 共同特点：功能成熟、生态完善、定价 $10-40/月
- 但：需要国际网络环境，学校无法直接使用
- **配图**：世界地图标注各工具出品国家
- **标注**："认知补全，非实践工具"

---

## 第 5 页：国外工具详解 — Cursor

- 出品方：Anysphere（美国）
- 定位：国外 AI IDE 标杆，开发者首选
- 核心模式：Tab 补全 + Chat + Composer 多文件编辑
- 定价：$20-40/月
- 一句话：目前全球口碑最好的 AI 编程 IDE
- **配图**：Cursor 界面截图（标注核心功能区）

---

## 第 6 页：国外工具详解 — Windsurf & Copilot

- **Windsurf**：Codeium 出品，Cascade 级联 Agent 模式，$15-30/月
  - 特点：AI 自主规划执行，像"实习生"一样工作
- **GitHub Copilot**：微软出品，最广泛使用的 AI 编程助手，$10/月
  - 特点：实时代码补全，最"无感"的 AI 体验
- **配图**：Windsurf Cascade 流程图 + Copilot 补全效果截图

---

## 第 7 页：国外工具详解 — Claude Code & Zed AI

- **Claude Code**：Anthropic 出品，命令行 AI 助手
  - 特点：长上下文理解极强，适合大型项目重构
  - 适合：喜欢终端操作的开发者
- **Zed AI**：Rust 编写的高性能编辑器
  - 特点：启动速度极快，原生多人协作
  - 适合：追求极致性能的开发者
- **配图**：Claude Code 终端界面 + Zed 编辑器界面

---

## 第 8 页：国内 AI 编程工具概览

- 国内替代方案：Trae / CodeBuddy / 通义灵码
- 共同特点：国内直连、中文优化、部分免费
- 核心认知：功能快速追赶，生态持续建设中
- **配图**：国内工具 logo 并列展示
- **对比表占位**：国外 vs 国内关键维度对比（功能/价格/网络/中文）

---

## 第 9 页：今日主角 — Trae 详解

- 出品方：字节跳动，2025/1/19 推出
- 定位：国内首个 AI 原生 IDE
- 核心优势：个人版免费、国内直连、三种模式覆盖全场景
- 支持模型：豆包 / DeepSeek / GPT-5.1 / GPT-5.2 / Gemini-3-Pro
- 企业版：69 元/月/席位
- 官网：trae.cn
- **配图**：Trae 启动界面 + logo

---

## 第 10 页：Trae 三种模式对比

- **Builder 模式**：你描述需求，AI 从零构建 → 适合新手、快速原型
- **Chat 模式**：边写边问，AI 实时顾问 → 适合所有人、日常辅助
- **Solo 模式**：你审批，AI 自主执行 → 适合有经验者、复杂任务
- 今天重点：Builder 模式
- **配图**：三种模式对比图（参与度 vs AI 参与度矩阵）

---

## 第 11 页：全球工具选型决策树

- 编程新手？→ Trae Builder
- 有国际网络？
  - 预算足 → Cursor
  - 预算有限 → GitHub Copilot
- 无国际网络？→ Trae（免费、直连、够用）
- **配图**：决策树流程图（树状分支结构）
- **核心信息**：标准要知道，实践要会用

## 第 15 页：分组策略与角色分配

- 分组：5-6 人/组，共 5-6 组
- 原则：计算机 + 非计算机混合、性别平衡
- 角色：
  - 组长（1人）：进度管理
  - QClaw 负责人（1人）：信息搜集
  - WorkBuddy 负责人（1人）：数据处理
  - Trae 负责人（1-2人）：开发实现
  - 产品/测试（1人）：需求与质量
- **配图**：团队角色分工图（圆形或矩阵布局）

---

## 第 16 页：需求拆解三步法

- 第一步：用户故事（Who + What + Why）
- 第二步：功能列表（P0 必须 / P1 加分 / P2 理想）
- 第三步：技术方案（输入 → 处理 → 输出）
- 原则：先完成，再完美
- **配图**：三步法流程图（箭头串联）

---

## 第 17 页：跨工具数据流转

- QClaw 模块：搜集 → 输出 raw_data.json
- WorkBuddy 模块：处理 → 输出 summary.json
- Trae 模块：展示 → 网页 Demo
- 关键：统一 JSON 格式（字段名一致）
- 技巧：模块未完成时，先用 mock 数据联调
- **配图**：数据流转示意图（三个模块 + 箭头 + 数据格式示例）
