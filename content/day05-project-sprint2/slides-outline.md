## 第 4 页：Trae 调试三板斧

- **终端错误**：从下往上读堆栈，最后一行是线索
- **浏览器控制台（F12）**：看红色报错、看 Network 状态码

- **核心技巧**：报错直接复制给 Builder，让 AI 帮你解读

## 第 5 页：API 接入 —— 给你的项目加上"实时数据"

- 什么是 API：数据外卖接口，请求 → 返回
- 教学用免费 API（国内直连）：
  - 和风天气：https://dev.qweather.com/（需注册 Key）
  - 聚合数据：https://www.juhe.cn/（新闻、历史上的今天）
  - 随机名言：https://v1.hitokoto.cn/（无需 Key，即开即用）
- Builder 模式一句话接入："调用 XX API 获取 XX 数据显示在 XX 位置"

## 第 6 页：多文件协作 —— 项目不是只有一个文件

- 标准项目结构：HTML（结构）+ CSS（样式）+ JS（逻辑）+ data（数据）
- Builder 自动管理：跨文件修改、自动引用、全局搜索
- 命名规范：用 `weather.js` 而不是 `1.js`
- 一个文件一个职责，API 调用单独封装

## 第 7 页：统一难度项目开发 —— 三选一

- **基础版**：天气查询小助手（输入城市 → 显示温度天气）
- **进阶版**：每日资讯卡片（自动展示新闻 → 点击展开摘要）
- **实战版**：农产品信息展示台（分类筛选 → 详情展示，与课程主题结合）

## 第 8 页：常见 Builder 报错速查（上）

- "Failed to fetch" → 检查 API 地址和网络
- "API Key invalid" → 确认 Key 正确、无空格
- "Cannot read property of undefined" → 数据结构不对，先 console.log 看返回
- 页面空白 → F12 看 Console 红色报错，检查元素 ID 是否匹配

---

## 第 9 页：常见 Builder 报错速查（下）

- "Module not found" → 检查 import 路径，改用原生 JS
- 样式不生效 → 检查 CSS 文件是否正确引入，选择器是否匹配
- Builder 一直"思考中" → 简化 Prompt，一次一个需求
- 保存失败 → Ctrl+S 手动保存，必要时重启 Trae

## 第 10 页：下午核心 —— 三工具联调流程图

- **数据流全景**：QClaw 搜集 → WorkBuddy 处理 → Trae 展示
- **联调四关**：
  1. QClaw 输出自检（格式、编码、路径）
  2. WorkBuddy 处理自检（读取、清洗、转换）
  3. Trae 展示自检（读取、渲染、交互）
  4. 端到端全流程测试（新数据从搜集到展示 < 5 分钟）
- **问题定位**：先看数据文件 → 再看中间结果 → 最后看前端

---

## 第 11 页：测试三级跳

- **功能测试**：每个功能点至少测一次，输入框测正常/空值/特殊字符
- **集成测试**：三模块协同走一遍，异常流程也要测
- **用户验收测试（UAT）**：找另一组当用户，3 分钟无指导完成 = 优秀
- 测试不是找茬，是确保明天路演不翻车

---

## 第 12 页：Bug 修复优先级

- **P0 阻断性**：核心功能不可用、程序崩溃 → 立即修，15 分钟修不好就降级
- **P1 功能性**：结果不对、部分场景失效 → 尽量修，太复杂就记为已知问题
- **P2 体验性**：颜色不好看、加载稍慢 → 有余力再修，文档注明即可
- 原则：不要在小问题上纠结半小时，先保主流程

---

## 第 13 页：项目文档模板展示

- **README 标准结构**：项目简介 → 功能特性 → 技术方案 → 安装运行 → 项目结构 → 使用说明 → 团队成员
- **技术方案文档**：需求背景 → 总体架构 → 模块设计 → 关键决策 → 测试方案 → 风险评估
- **使用说明写法**：一步一动作、配截图、标预期结果、含故障排除
- 文档是写给人看的，假设读者完全不懂技术

## 设计建议

### 配色方案
- 主色：Trae 品牌蓝（#1E90FF 或提取 Trae 官网主色）
- 辅色：QClaw 绿、WorkBuddy 橙，体现三工具协作
- 背景：浅色底（白/浅灰），确保投影清晰可见

### 重点页面视觉强化
- **第 5 页（API 接入）**：配一张 API 调用示意图（请求 → 服务器 → 返回数据 → 页面展示）
- **第 10 页（联调流程图）**：必须画流程图，用箭头清晰展示数据流
- **第 13 页（文档模板）**：截图展示 README 模板的关键区块，用红框标注填空位置

### 动画建议
- 流程图页面使用"擦除"或"飞入"动画，按数据流顺序逐步显示
- 报错速查页面使用"淡入"，避免信息过载
- 避免花哨转场，保持专业感

### 备注页（给讲师看，不投影）
- 每页底部添加讲师话术提示（对应详细授课文档中的加粗话术）
- 第 7 页备注：如果学生大面积卡住，统一降级到"农产品信息展示台"
- 第 10 页备注：联调阻塞超 30 分钟，建议改用纯本地数据方案
