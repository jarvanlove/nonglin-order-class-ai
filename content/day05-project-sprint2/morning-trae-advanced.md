## 二、Trae 调试方法（09:10 - 09:40，30 分钟）

### 2.1 错误查看三板斧

**讲师话术**：
> "写代码不出错是不可能的，关键是出错之后怎么快速定位。Trae 的 Builder 模式虽然帮你写了很多代码，但运行时报错是常事。"

1. **终端错误信息（Terminal）**
   - 位置：Trae 底部面板 → Terminal 标签
   - 关键动作：报错后先看最后一行，从下往上读错误堆栈
   - 常见格式：`Error: xxx at line 15 in file app.js`
   - Builder 模式下，直接把报错信息复制到 Chat 框，让 AI 解释并修复

2. **浏览器控制台（F12）**
   - 适用场景：前端网页项目
   - 打开方式：按 F12 → Console 标签
   - 重点查看红色报错和黄色警告
   - 网络请求失败看 Network 标签，状态码 404/500 是常见问题

3. **Trae 内置问题面板（Problems）**
   - 位置：底部面板 → Problems 标签
   - 显示语法错误、类型不匹配等静态检查问题
   - 红色波浪线 = 必须修，黄色波浪线 = 建议修

### 2.2 日志分析技巧

**讲师话术**：
> "打印日志是程序员最古老的调试方法，也是最可靠的。"

- 在 JavaScript/Python 代码中插入 `console.log()` 或 `print()`
- 关键位置必打日志：函数入口、数据返回处、条件分支
- Builder 模式下，可以告诉 Trae："在关键步骤加上日志输出，方便调试"
- 日志要打印"变量名 + 值"，不要只打印"到这里了"

**示例**：
```javascript
// 不好的日志
console.log("ok");

// 好的日志
console.log("API返回数据:", response.data);
console.log("用户输入:", userInput, "长度:", userInput.length);
```

### 2.3 回退操作：代码时光机

**讲师话术**：
> "Builder 模式有时候会改坏你的代码，或者越改越乱。这时候不要慌，Trae 有内置的版本回溯。"

- **撤销上一步修改**：Ctrl+Z（Windows）/ Cmd+Z（Mac），可连续撤销多步
- **查看文件历史**：右键文件 → "查看历史" 或 "Compare with Previous"
- **Git 回退（如果已初始化仓库）**：
  ```bash
  git log --oneline          # 查看提交历史
  git reset --hard HEAD~1    # 回退到上一次提交（慎用，会丢失未提交改动）
  git checkout -- filename   # 恢复单个文件到上次提交状态
  ```
- ** Builder 模式专属**：如果对一轮 Builder 修改不满意，可以在 Chat 中说"回退刚才的修改"，Trae 会尝试撤销

## 三、API 接入基础（09:40 - 10:10，30 分钟）

### 3.1 什么是 API？为什么项目需要它？

**讲师话术**：
> "你的项目不可能所有数据都存在本地，比如要显示天气、新闻、农产品价格，这些实时数据必须从互联网获取。API 就是'数据外卖接口'——你发一个请求，对方返回数据。"

API 调用三要素：
1. **地址（URL）**：去哪里取数据
2. **方法（Method）**：GET（获取）/ POST（提交）
3. **参数（Params）**：告诉对方你要什么数据

### 3.2 国内可直接使用的免费 API（无需 VPN）

以下 API 均在国内网络环境下可直接访问，适合教学使用：

#### 示例 1：天气查询 API（和风天气）

**注册与获取 Key**：
- 访问 https://dev.qweather.com/
- 免费注册，创建项目获取 API Key
- 免费版有每日调用次数限制，教学够用

**调用示例**：
```javascript
// 获取北京实时天气
const url = 'https://devapi.qweather.com/v7/weather/now?location=101010100&key=你的KEY';

fetch(url)
  .then(response => response.json())
  .then(data => {
    console.log('温度:', data.now.temp);
    console.log('天气:', data.now.text);
  })
  .catch(error => console.error('请求失败:', error));
```

#### 示例 2：新闻 API（NewsAPI 国内镜像或聚合数据）

**聚合数据（国内平台）**：
- 访问 https://www.juhe.cn/
- 提供新闻头条、历史上的今天等免费接口
- 注册后获取 AppKey

**调用示例**：
```javascript
const url = 'http://v.juhe.cn/toutiao/index?type=top&key=你的APPKEY';

fetch(url)
  .then(response => response.json())
  .then(data => {
    const newsList = data.result.data;
    newsList.forEach(item => {
      console.log(item.title);
    });
  });
```

#### 示例 3：随机名言 API（无需 Key，即开即用）

**适用于教学演示，无需注册**：
```javascript
// 随机中文名言
fetch('https://v1.hitokoto.cn/')
  .then(response => response.json())
  .then(data => {
    console.log('名言:', data.hitokoto);
    console.log('出处:', data.from);
  });
```

#### 示例 4：IP 地址查询（无需 Key）

```javascript
fetch('https://api.ipify.org?format=json')
  .then(response => response.json())
  .then(data => {
    console.log('当前IP:', data.ip);
  });
```

### 3.3 在 Trae Builder 模式中使用 API

**讲师话术**：
> "你不需要手写这些代码。在 Builder 模式下，直接描述你的需求，Trae 会自动帮你写 API 调用代码。"

**推荐 Prompt 模板**：
```

API地址是 [URL]，

把获取到的数据显示在页面的 [位置] 上。
如果请求失败，显示"数据加载失败，请稍后重试"。
```

**实战演示**：
1. 打开 Trae Builder 模式
2. 输入："调用和风天气 API 获取当前城市温度，显示在页面顶部"

4. 把示例中的 `你的KEY` 替换为真实 Key

## 四、多文件协作（10:10 - 10:25，15 分钟）

### 4.1 Trae 如何管理项目中的多个文件

**讲师话术**：
> "一个真正的项目不可能只有一个文件。今天你们要做的模块，至少会涉及 HTML、CSS、JavaScript 三个文件，可能还有图片和配置文件。"

**文件组织原则**：
```
my-project/
├── index.html          # 入口页面
├── css/
│   └── style.css       # 样式文件
├── js/
│   └── main.js         # 主逻辑
│   └── api.js          # API 调用封装
├── images/
│   └── logo.png
└── README.md           # 项目说明
```

### 4.2 Builder 模式下的多文件操作

- **自动创建文件**：在 Builder 中描述"创建一个样式文件来美化页面"，Trae 会自动创建 `.css` 文件
- **跨文件修改**：Trae 会分析文件之间的引用关系，修改 HTML 中引用的 JS 文件时，会自动同步
- **查看文件关系**：左侧资源管理器显示文件树，点击文件可快速切换
- **全局搜索**：Ctrl+Shift+F 可以搜索整个项目中的关键词，快速定位代码

### 4.3 多文件协作最佳实践

1. **一个文件一个职责**：HTML 管结构、CSS 管样式、JS 管逻辑
2. **API 调用单独封装**：把所有 `fetch` 请求集中放在一个 `api.js` 中
3. **命名要规范**：不要用 `1.js`、`2.js`，用 `weather.js`、`user.js` 这种有意义的名称
4. **定期保存**：Ctrl+S 保存当前文件，Trae 有自动保存但手动保存更保险

## 七、常见 Builder 模式报错及解决方法

### 错误 1："Failed to fetch" 或 "Network Error"

**原因**：网络请求失败，可能是 API 地址错误、网络不通、CORS 跨域限制
**解决**：
- 检查 API 地址是否拼写正确
- 在浏览器 F12 → Network 中查看具体请求状态
- 如果是 CORS 问题，尝试在 Trae 中让 Builder 添加代理或使用支持 CORS 的 API

### 错误 2："API Key invalid" 或 "认证失败"

**原因**：API Key 未填写、填写错误、或 Key 已过期
**解决**：
- 确认已注册并获取正确的 Key
- 检查 Key 是否拼写完整，没有多余空格
- 确认使用的是开发版 Key 还是生产版 Key

### 错误 3："Cannot read property 'xxx' of undefined"

**原因**：访问了一个未定义对象的属性，通常是 API 返回数据结构与预期不符
**解决**：
- 在 `fetch` 的 `.then` 中先 `console.log(data)` 查看实际返回结构
- 给 Builder 描述实际返回的数据结构，让它调整代码
- 添加空值检查：`if (data && data.now) { ... }`

### 错误 4：Builder 生成的代码运行后页面空白

**原因**：JavaScript 执行出错导致后续代码未执行，或 DOM 元素未正确获取
**解决**：
- 打开 F12 → Console 查看红色报错
- 检查 HTML 中元素 ID 是否与 JS 中 `getElementById` 一致
- 确认 JS 文件是否在 HTML 中正确引用

### 错误 5："Module not found" 或 "找不到模块"

**原因**：Builder 引入了不存在的文件或库
**解决**：
- 检查 `import` 或 `script src` 的路径是否正确
- 如果是外部库（如 Vue/React），确认是否需要 CDN 引入
- 让 Builder 改为使用原生 JavaScript，不依赖外部框架

### 错误 6：样式不生效，页面很丑

**原因**：CSS 文件未正确引入，或选择器不匹配
**解决**：
- 检查 HTML `<head>` 中是否有 `<link rel="stylesheet" href="css/style.css">`
- 检查 CSS 选择器是否与 HTML 元素对应
- 在 F12 → Elements 中查看元素实际应用的样式

### 错误 7：Builder 模式一直"思考中"不生成代码

**原因**：网络延迟、Trae 服务繁忙、Prompt 太长或太复杂
**解决**：
- 等待 1-2 分钟，不要频繁点击
- 简化 Prompt，一次只提一个需求
- 如果长时间无响应，取消后重新发送
- 切换到 Chat 模式先讨论方案，再回 Builder 生成

### 错误 8：代码生成后保存失败或文件未更新

**原因**：Trae 文件系统同步延迟，或权限问题
**解决**：
- 手动按 Ctrl+S 保存
- 检查左侧文件树是否显示新文件
- 如果 Builder 提示"已应用"但文件未变，尝试关闭文件重新打开
- 重启 Trae（保存好现有代码后再重启）
