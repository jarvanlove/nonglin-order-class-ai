# Day 1 上午理论讲义：AI 全景认知与全球工具生态

**授课对象**：30 名应届毕业生（计算机 + 非计算机混编）
**授课时长**：3 小时（09:00 - 12:00）
**授课风格**：严肃学术 + 工程实战 + 轻松科普，三者兼顾

## 一、破冰环节（09:00 - 09:30，30 分钟）

### 1.1 讲师开场白

> **【讲师话术】**
> 各位同学早上好。我是你们这 6 天 AI 工具实训课程的主讲老师。先问大家一个问题：你们当中有多少人，在过去一年里用过 ChatGPT、文心一言、Kimi 这类 AI 聊天工具？请举手。
>
> （等待举手，观察比例）
>
> 好，我看到大部分同学都举了手。那第二个问题：你们当中有多少人，觉得这些工具"有时候很神奇，有时候又很傻"——比如让它写个东西写得挺好，但让它处理一个稍微复杂点的任务，它就懵了？
>
> （等待回应，微笑）
>
> 这就是我们今天要解决的问题。过去两年，AI 已经从"聊天玩具"进化成了"数字员工"——它能帮你搜集信息、整理资料、写代码、做表格，甚至帮你自动发微信。6 天之后，你们每个人手里都会有一套"上岗即用"的国产 AI 工具技能。不管你们毕业后去互联网公司、考公务员、还是回农林系统做技术推广，这些工具都能让你们的工作效率提升一个量级。
>
> 先说一下课程安排：6 天，每天 6.5 小时，上午讲理论，下午动手练。第 6 天各组要上台展示一个完整的跨工具项目。考核方式很简单：出勤 + 课堂参与 + 项目成果。没有考试，但每天都有产出，产出的质量就是你们的成绩。

>
> 我们从左边第一排开始。

**讲师记录要点**：
- 记录非计算机学生的比例和具体专业（农学、林学、园艺、经管等）
- 记录学生提出的典型需求（写论文、做表格、搜资料、写代码等）
- 在后续讲课中穿插引用学生的实际需求作为案例

### 1.3 课程总览说明（10 分钟）

> **【讲师话术】**
> 我快速过一下 6 天的安排，大家心里有个地图：
>
> - **Day 1（今天）**：建立全球 AI 认知地图，上手第一个国产 Agent 工具 QClaw
> - **Day 2**：深入理解 Agent 原理，掌握 WorkBuddy 办公自动化
> - **Day 3**：了解全球 AI 编程工具生态，上手 Trae，启动跨工具项目

## 二、AI 发展简史：从 GPT-3 到 Agent 范式（09:30 - 10:30，60 分钟）

### 2.1 时间线总览

> **【讲师话术】**
> 要讲清楚今天的 AI 工具为什么长这样，我们必须快速回顾一下过去 5 年的关键节点。我不是要给你们上历史课，而是要帮你们理解一个核心趋势：AI 正在从"你问我答"的聊天模式，进化成"你交代任务、它自主执行"的 Agent 模式。这个趋势决定了你们未来 3-5 年的职业竞争力。

#### 关键里程碑时间线

| 时间 | 事件 | 意义 |
|------|------|------|
| 2020 年 6 月 | OpenAI 发布 GPT-3（1750 亿参数） | 第一次证明"大模型"可以生成连贯、有用的文本，AI 从实验室走向应用 |
| 2022 年 11 月 | OpenAI 发布 ChatGPT | 全球现象级产品，5 天用户破百万，让普通人第一次直观感受到 AI 的能力 |
| 2023 年 3 月 | GPT-4 发布，多模态能力大幅提升 | AI 开始能看懂图片、理解复杂指令，企业级应用爆发 |
| 2023 年 6 月 | OpenClaw 开源（GitHub 迅速获得 10K+ Stars） | 开源社区第一次有了"可扩展的 AI Agent 框架"，Agent 概念从论文走向工程 |
| 2023 年 10 月 | 国内大模型牌照发放，文心一言、通义千问等正式面向公众 | 国产 AI 工具开始形成独立生态，不再完全依赖国外技术 |
| 2024 年 3 月 | Claude 3 发布，代码能力大幅提升；Cursor IDE 用户激增 | AI 编程工具成为开发者标配，"AI 原生开发"概念兴起 |
| 2024 年 6 月 | OpenClaw GitHub Stars 突破 20 万，ClawHub Skills 市场形成 | 开源 Agent 生态成熟，第三方开发者可以为 Agent 开发"技能插件" |
| 2025 年 1 月 | 各类 Agent 产品爆发：Devin（AI 软件工程师）、各类浏览器 Agent、办公 Agent | "Agent"取代"大模型"成为行业最热关键词 |
| 2025 年 6 月 | 腾讯发布 QClaw，基于 OpenClaw 架构本地化适配，微信直连 | 国产 Agent 工具开始面向普通用户，零门槛使用 |
| 2026 年 4 月 | 今天 | Agent 工具已经成为职场基础技能，就像 2010 年的 Word 和 Excel |

> **【讲师话术】**
> 注意看这条时间线的后半段：2024 年之后，行业的关注点从"模型有多大"转向了"Agent 能做什么"。这是一个本质性的转变。大模型是"大脑"，Agent 是"完整的人"——有大脑、有手脚、能记忆、能调用工具。

### 2.2 范式转移：从 Chat 到 Agent

> **【讲师话术】**
> 我用一个比喻帮大家理解这个转变：
>
> **Chat 模式** = 你请了一个非常聪明的顾问，坐在你对面。你问他什么，他答什么。但他不会主动做事，不会记住你们一个月前的对话，也不会自己去查资料、发邮件、写代码。他只是一个"会说话的知识库"。
>
> **Agent 模式** = 你请了一个数字员工。你交代任务："搜集近 3 年农林领域 AI 应用案例，整理成结构化笔记，发到我微信。"然后它自己去搜索、去筛选、去整理、去格式化，最后直接把结果推送到你的微信。你不需要一步步教它怎么做，只需要交代目标和验收标准。
>
> 这就是我们今天要学的核心——不是怎么跟 AI 聊天，而是怎么指挥 AI 干活。

### 2.3 Agent 的核心特征（四要素）

1. **自主性（Autonomy）**：给定目标后，能自主规划步骤并执行，不需要人类每一步指令
2. **记忆性（Memory）**：能记住上下文、记住用户偏好、记住历史任务，越用越懂你
3. **工具调用（Tool Use）**：能调用外部工具——搜索引擎、数据库、微信、Excel、代码执行环境
4. **持续运行（Persistence）**：不需要你一直盯着，可以在后台持续工作，完成后主动通知你

> **【讲师话术】**
> 这四个特征，你们今天就会在 QClaw 里亲身体验到。现在先记住这个框架，下午实践的时候我会提醒你们对照。

## 三、全球 AI 工具生态对比（10:45 - 12:00，75 分钟）

> **【讲师话术】**
> 接下来这张图，是你们今天最重要的认知产出——全球 AI 工具生态地图。我会分国外和国内两部分讲。再次强调：国外工具今天只讲认知，不操作；所有实践都用国产工具。

### 3.1 国外主流 AI 工具（认知补全）

> **【讲师话术】**
> 为什么我要讲国外工具？三个原因：第一，你们毕业后可能去外企、互联网公司、或者出国深造，那时候你们需要知道这些工具；第二，国产工具的很多设计思路源自国外开源项目，理解源头有助于理解国产工具；第三，面试的时候，如果面试官问"你用过哪些 AI 工具"，你只回答国产的，可能会显得视野不够宽。

#### 工具定位说明

| 工具名称 | 定位 | 核心能力 | 适用场景 |
|----------|------|----------|----------|
| **Claude Code**（Anthropic） | AI 编程助手，命令行交互 | 代码生成、代码审查、重构、多文件协作 | 软件开发、代码维护、技术方案设计 |
| **Cursor**（Anysphere） | AI 原生 IDE（集成开发环境） | 在 VS Code 基础上深度集成 AI，支持 Composer 多文件编辑、Chat 对话编程 | 全栈开发、项目构建、AI 辅助编程 |
| **GitHub Copilot**（Microsoft） | 代码补全与生成插件 | 实时代码建议、注释生成代码、单元测试生成 | 日常编码加速、学习新语言 |
| **ChatGPT**（OpenAI） | 通用对话 AI | 文本生成、问答、翻译、创意写作、简单代码 | 通用知识查询、文案写作、学习辅导 |
| **Gemini**（Google） | 多模态通用 AI | 文本、图片、音频、视频综合理解，长上下文（100万+ token） | 多媒体内容分析、长文档处理、跨模态任务 |
| **OpenClaw**（开源社区） | 开源 AI Agent 平台 | 支持 20+ 消息渠道接入，ClawHub 13,700+ Skills，可私有化部署 | 企业级 Agent 搭建、自定义自动化工作流 |

> **【讲师话术】**
> 我日常工作中用得最多的是 Claude Code 和 Cursor。Claude Code 适合在命令行里快速处理代码问题，Cursor 适合完整的项目开发。但这两个工具都需要国外网络环境，我们学校用不了。所以我们的实践会用国产替代方案。

#### 标注说明

> **【讲师话术】**
> 请大家记住一个原则：**国外工具 = 认知补全，国产工具 = 实践上手**。今天之后，你们要知道 Claude 和 Cursor 是什么、能做什么、在什么场景下用。但你们的手上功夫，要建立在 QClaw、WorkBuddy、Trae 这三个国产工具上。

### 3.3 国外 vs 国内工具对比总结

| 对比维度 | 国外工具（Claude/Cursor/Copilot 等） | 国内工具（QClaw/WorkBuddy/Trae 等） |
|----------|--------------------------------------|--------------------------------------|
| **网络要求** | 需要 VPN 或国外网络环境 | 纯国内网络可用 |
| **语言支持** | 英文优先，中文支持较好 | 中文原生优化 |
| **生态集成** | 与 GitHub、VS Code、Slack 等国际生态深度集成 | 与微信、企业微信、钉钉、飞书等国内生态集成 |
| **代码能力** | 整体领先，尤其复杂工程场景 | Trae Builder 模式在简单项目场景已足够 |
| **数据安全** | 数据可能出境，企业需谨慎评估 | 数据留在国内，符合国内合规要求 |
| **使用成本** | 多数收费（$20-40/月） | 大量免费额度（如 QClaw 4000 万 Token/天） |
| **本课程定位** | 仅认知，不实践 | 全部实践操作 |

> **【讲师话术】**
> 最后这张对比表，建议你们截图保存。毕业后进入职场，你们会面临工具选型的问题。选国外还是国产？没有标准答案，要看你的网络环境、数据合规要求、预算、以及具体场景。但前提是——你得知道有哪些选项，以及每个选项的优劣。这就是今天这 75 分钟的价值。

## 五、讲师备注

### 5.1 现场演示建议
- 国外工具演示：提前打开 Claude Code / Cursor / ChatGPT 界面，展示外观和基本交互，但不在学生电脑上操作
- 国内工具演示：QClaw 官网（qclaw.qq.com）可在教室网络下正常访问，可现场展示首页和下载页面
- 建议准备一张"工具生态地图"高清图，投屏展示

### 5.2 非计算机学生关注要点
- 强调"不需要写代码"：QClaw 和 WorkBuddy 都是自然语言交互
- 用农林领域案例贴近学生专业：智慧农业、病虫害识别、林业遥感、农产品价格预测等
- 避免过度技术术语：用"数字员工"代替"自主智能体"，用"技能插件"代替"Function Calling"

### 5.3 常见学生提问预设
- **Q：为什么学校不能用国外工具？** A：网络管理政策，涉及数据安全和合规。理解并遵守，同时通过理论课补全认知。
- **Q：国产工具真的够用吗？** A：对于 80% 的日常办公和信息处理场景，完全够用。剩下 20% 的复杂工程场景，等你们有权限用国外工具时再补充。
- **Q：6 天能学到什么程度？** A：能独立完成信息搜集、文档处理、简单项目开发，能组合三个工具完成一个完整项目。这是"上岗即用"的水平，不是"专家"水平。
