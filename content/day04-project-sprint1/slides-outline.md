## 第 4 页：常见阻塞 Top 8（上）

- QClaw 微信连接失败：先查版本，再切热点，最后重扫码
- WorkBuddy 文件权限不足：右键"以管理员身份运行"，检查隐私设置
- Trae Builder 报错：缺依赖就 `npm install`，端口占用就 `kill-port`
- 数据格式不匹配：QClaw 明确指定 JSON 字段，WorkBuddy 配置字段映射
- 配图需求：4 个错误图标 + 对应解决方案的简洁图示（如 WiFi 图标、盾牌图标、终端图标、文件图标）

## 第 5 页：常见阻塞 Top 8（下）

## 第 6 页：QClaw 进阶 —— Skills 调用

- Skills = 给 QClaw 装"专业插件"
- 去 ClawHub 搜索 → 点击"添加" → 对话中 `@Skill名` 调用
- 同一个任务，用 Skill 输出更结构化、更专业
- 学生任务：为项目选一个 Skill，完成一次实际采集
- 配图需求：ClawHub 界面截图（打码处理）+ Skill 调用前后对比示意图

## 第 8 页：WorkBuddy 进阶 —— 多 Agent 流水线

- Agent 1（分类员）：监控输入文件夹，按类型分类
- Agent 2（处理员）：处理分类后的文件，清洗格式化
- Agent 3（报告员）：汇总数据，生成最终文档
- 关键：触发条件互斥、输出目录分离、串行执行
- 配图需求：流水线流程图（3 个工人图标 + 箭头 + 文件夹图标）

---

## 第 9 页：WorkBuddy 进阶 —— 模板配置

- 步骤 1：Word/Excel 中设计模板，用 `{{字段名}}` 标注填充位
- 步骤 2：WorkBuddy 导入模板，映射数据源
- 步骤 3：测试跑通，微调格式（日期、数字、空值）
- 价值：一次设计，永久复用，一键生成
- 配图需求：模板设计示意图（Word 文档截图，高亮 `{{字段}}` 标记）

## 附录：幻灯片制作备注

### 字体建议
- 标题：思源黑体 Bold，32-36pt
- 正文：思源黑体 Regular，20-24pt
- 代码/JSON：JetBrains Mono 或 Consolas，18pt，行距 1.5

### 配色方案
- 背景：纯白 #FFFFFF
- 主文字：深灰 #333333
- 强调色：课程主色（建议蓝色 #2563EB 或绿色 #059669）
- 警告/阻塞：橙色 #EA580C
- 错误：红色 #DC2626
- 通过/成功：绿色 #16A34A

### 动画建议
- 项目日幻灯片减少动画，以信息密度为主
- 如需动画，仅用"淡入"和"擦除"，避免花哨效果
- 第 4-5 页（阻塞问题）可逐条出现，配合讲师讲解节奏

### 截图/配图来源
- QClaw/WorkBuddy/Trae 界面截图：由讲师在课前准备，确保版本与教学一致
- 流程图/示意图：可用 draw.io 或 Figma 绘制，导出 PNG/SVG
- Mock 数据示例：直接复制 afternoon-dev.md 中的 JSON，放大字号投影
