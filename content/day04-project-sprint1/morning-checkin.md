## 二、常见阻塞问题预判与解答（站会中及站会后使用）

以下 8 个问题是前三天教学中最高频出现的阻塞，讲师和助教必须提前熟悉答案。

### 问题 1：QClaw 微信连接失败 / 扫码后无响应

**现象**：学生扫码绑定微信后，QClaw 显示"连接中"或收不到消息。

**根因与解决**：
1. 检查微信版本：必须更新到最新版（设置 > 关于微信 > 检查更新）
2. 检查网络：学校网络可能屏蔽了 QClaw 的 WebSocket 端口，尝试切换手机热点
3. 重新扫码：在 QClaw 设置中点击"解除绑定"，重新扫码
4. 防火墙问题：Windows  Defender 可能拦截，暂时关闭专用网络防火墙测试

**话术提示**：**"先别急着重装，90% 的微信连接问题都是网络或版本问题，按这个顺序排查，3 分钟就能定位。"**

### 问题 2：WorkBuddy 文件权限不足 / 无法读取桌面文件

**现象**：WorkBuddy 提示"无法访问该文件夹"或文件列表为空。

**根因与解决**：
1. Windows：右键 WorkBuddy 图标 >"以管理员身份运行"
2. macOS：系统设置 > 隐私与安全 > 文件和文件夹 > 给 WorkBuddy 开启"桌面"和"文档"权限
3. 路径问题：WorkBuddy 默认只能访问用户目录下的文件，不要把素材放到 C 盘根目录或系统目录
4. 中文路径：部分旧版本对中文路径支持不好，尝试把文件夹改成英文名称

**话术提示**：**"这不是 bug，是操作系统在保护你。给权限就行，但记住——生产环境里不要随便给任何软件管理员权限。"**

### 问题 3：Trae Builder 模式报错 / 生成的代码运行失败

**现象**：Builder 生成代码后，终端显示红色报错，学生不知道怎么办。

**根因与解决**：
1. 依赖缺失：Builder 有时忘了装依赖，手动在终端运行 `npm install`（Node 项目）或 `pip install -r requirements.txt`（Python 项目）
2. 端口占用：提示 `EADDRINUSE`，说明 3000 或 8080 端口被别的程序占了，在 Trae 终端运行 `npx kill-port 3000` 或换端口
3. 语法错误：Builder 生成的代码偶尔有语法问题，把报错信息复制到 Chat 模式，让 AI 修复
4. 文件路径错误：Windows 和 macOS 路径斜杠不同，Builder 有时混用，手动改成反斜杠 `\`

**话术提示**：**"看到报错别慌，把报错信息原样复制给 Trae 的 Chat 模式，它比你还想修好。"**

### 问题 4：跨工具数据格式不匹配（QClaw 输出 WorkBuddy 读不懂）

**现象**：QClaw 采集的数据是文本/JSON，WorkBuddy 处理时格式错乱。

**根因与解决**：
1. QClaw 输出时明确指定格式：在指令末尾加"请以标准 JSON 格式输出，字段包括：title, content, source, date"
2. WorkBuddy 读取前先做格式校验：用 WorkBuddy 的"数据清洗"功能，设置字段映射规则
3. 中间文件：让 QClaw 把结果写入一个 `.json` 文件到桌面，WorkBuddy 从桌面读取该文件
4. 编码问题：确保文件保存为 UTF-8，不要 ANSI 或 GBK

**话术提示**：**"两个工具之间不会自动'心有灵犀'，你必须当翻译——告诉 QClaw 按什么格式输出，告诉 WorkBuddy 按什么格式读取。"**

### 问题 5：QClaw Skills 调用失败 / 提示"未找到该 Skill"

**现象**：学生在 QClaw 中输入 Skill 名称，提示不存在或调用失败。

**根因与解决**：
1. 检查 Skill 名称拼写：区分大小写，建议复制粘贴不要手打
2. 检查 Skill 是否已安装：进入 QClaw 的 Skills 市场，确认该 Skill 已添加
3. 参数格式错误：有些 Skill 需要特定参数格式，查看 Skill 说明文档
4. 网络问题：Skills 调用需要联网，学校网络不稳定时可能超时，重试即可

**话术提示**：**"Skills 就像微信小程序，你得先'安装'才能'打开'。先去 ClawHub 点一下'添加'，回来再用。"**

### 问题 6：WorkBuddy 多 Agent 设置后任务冲突 / 重复执行

**现象**：配置了两个 Agent 后，同一个文件被处理两次，或两个 Agent 互相覆盖结果。

**根因与解决**：
1. 检查触发条件：两个 Agent 的触发条件不能重叠，比如一个监控"桌面/输入"文件夹，另一个监控"桌面/输出"文件夹
2. 设置执行顺序：在 WorkBuddy 的"工作流"设置中，明确 Agent A 执行完后再触发 Agent B
3. 输出目录分离：每个 Agent 的输出必须指向不同文件夹，避免覆盖
4. 日志检查：WorkBuddy 有执行日志，查看哪一步重复了，调整条件

**话术提示**：**"多 Agent 不是人多力量大，是分工要明确。就像流水线，每个人的工位不能重叠。"**

### 问题 7：Trae 项目文件太多，Builder 模式"失忆"

**现象**：项目文件超过 10 个后，Builder 修改 A 文件时把 B 文件改坏了。

**根因与解决**：
1. 分模块开发：不要一次让 Builder 处理整个项目，一次只聚焦 1-2 个文件
2. 使用 Chat 模式做局部修改：小改动用 Chat，大重构用 Builder
3. 及时提交：每完成一个功能点，在 Trae 左侧源代码管理面板点击"+"暂存，写一句提交说明
4. 备份关键文件：修改前手动复制一份 `.bak`，改坏了能回退

**话术提示**：**"Builder 的'记忆力'有限，文件太多它会'顾此失彼'。大事用 Builder，小事用 Chat，这是 Trae 的生存法则。"**

### 问题 8：项目需求过大，Day 4 上午还没做完基础框架

**现象**：组内还在争论做什么、怎么做，没有可运行的代码或流程。

**根因与解决**：
1. 立即裁剪需求：砍掉所有"锦上添花"的功能，只保留"没有它项目就不成立"的核心功能
2. 参考 MVP 标准（见下午文档）：信息处理模块能跑通 + 办公自动化模块能输出 = 及格
3. 讲师介入决策：如果组内无法达成一致，讲师直接指定当天必须完成的功能点
4. 换分工：让擅长沟通的学生专门负责 QClaw/WorkBuddy 的配置，让有代码基础的学生负责 Trae

**话术提示**：**"现在不是追求完美的时候，是先让火车跑起来。哪怕是个手摇的火车，也比图纸强。"**

## 三、QClaw 进阶技巧集中培训（10:15-11:00，45 分钟）

### 3.1 Skills 调用：让 QClaw 具备专业能力

**什么是 Skills**：QClaw 的 Skills 是可插拔的能力模块，相当于给 QClaw 装"专业插件"。

**教学演示步骤**：
1. 打开 QClaw，点击左侧"Skills"图标进入 ClawHub
2. 搜索并添加一个实用 Skill（如"网页内容提取"或"PDF 解析"）
3. 在对话框中调用：`@网页内容提取 https://example.com`，观察 Skill 如何接管任务
4. 对比不用 Skill 的结果：同一个任务，用 Skill 的输出更结构化

**学生练习任务**：
- 每组为自己的项目选一个相关 Skill（如做文献整理选"PDF 解析"，做新闻监控选"网页内容提取"）
- 用 Skill 完成一次实际采集，把结果保存到桌面 `qclaw-output.json`

**话术提示**：**"不用 Skill 的 QClaw 是个通才，用了 Skill 才是专家。你的项目需要什么专业能力，就去 ClawHub 找对应的 Skill。"**

### 3.2 ClawHub 生态接入：找到适合你项目的工具

**ClawHub 使用策略**：
1. 按场景筛选：ClawHub 有分类标签（办公、学术、开发、生活），不要漫无目的浏览
2. 看使用量排序：优先选"使用人数多 + 评分高"的 Skill，避免用冷门无人维护的
3. 组合使用：一个复杂任务可以串联多个 Skill，如"网页采集 → 内容摘要 → 翻译 → 导出 Excel"

**推荐学生项目高频 Skills（提前准备清单投影）**：

| 项目类型 | 推荐 Skill | 用途 |
|---------|-----------|------|
| 信息搜集类 | 网页内容提取、RSS 订阅 | 自动采集目标网站信息 |
| 文献整理类 | PDF 解析、学术搜索 | 提取论文核心内容 |
| 数据分析类 | CSV 处理、图表生成 | 把文本数据可视化 |
| 微信自动化 | 群消息监控、定时发送 | 自动回复和消息推送 |

**话术提示**：**"ClawHub 里有几百个 Skill，但你不需要都学会。就像去五金店，你知道自己要修什么，再去找对应的工具。"**

### 3.3 微信小程序进阶操作：从"能用"到"好用"

**QClaw 微信小程序的三个进阶用法**：

1. **快捷指令**：把常用指令保存为"快捷短语"，避免每次重复打字
   - 操作路径：小程序 > 我的 > 快捷短语 > 添加
   - 示例：保存"整理以下内容为标准 JSON，字段包括 title, summary, url"

2. **文件传输**：小程序支持直接发送文件给 QClaw 处理
   - 操作路径：聊天窗口点击"+" > 文件 > 选择本地文件
   - 适用场景：手机拍了照片/收到 PDF，直接发给 QClaw 分析

3. **群聊接入**：把 QClaw 拉进项目微信群，实现群消息自动处理
   - 操作路径：QClaw 主界面 > 微信绑定 > 选择群聊 > 开启"群消息处理"
   - 注意事项：群聊消息量大时设置关键词过滤，避免 QClaw 被刷屏

**话术提示**：**"微信小程序是 QClaw 的'移动指挥部'。你在食堂排队时都能用手机指挥 QClaw 干活，这才是 Agent 的真正价值。"**

## 四、WorkBuddy 进阶技巧集中培训（11:00-12:00，60 分钟）

### 4.1 多 Agent 并行设置：一条流水线，多个工人

**核心概念**：WorkBuddy 支持配置多个 Agent，每个 Agent 负责一个环节，形成自动化流水线。

**教学演示：搭建"文件整理流水线"**

步骤 1：创建 Agent 1（文件分类员）
- 触发条件：监控 `C:\Users\学生名\Desktop\待处理` 文件夹
- 任务：识别文件类型，移动到对应子文件夹（文档/图片/表格/其他）
- 输出：`C:\Users\学生名\Desktop\已分类`

步骤 2：创建 Agent 2（表格处理员）
- 触发条件：监控 `C:\Users\学生名\Desktop\已分类\表格` 文件夹
- 任务：打开 Excel 文件，自动格式化（加表头、调列宽、去重）
- 输出：`C:\Users\学生名\Desktop\已处理\表格`

步骤 3：创建 Agent 3（报告生成员）
- 触发条件：当 Agent 2 完成且输出文件夹有新文件
- 任务：汇总所有表格数据，生成一份汇总报告（Word/PDF）
- 输出：`C:\Users\学生名\Desktop\报告`

**关键配置要点**：
- 每个 Agent 的触发条件必须**互斥或有序**，避免同时触发
- 输出目录必须**分离**，不要两个 Agent 写到同一个文件夹
- 在"工作流"面板中开启"串行执行"，默认并行可能导致冲突

**学生练习任务**：
- 每组根据自己项目需求，设计至少 2 个 Agent 的协作流程
- 画出流程图（可用纸笔或 PPT），标注触发条件和数据流向

**话术提示**：**"单 Agent 是单打独斗，多 Agent 是团队协作。但团队要有队长——WorkBuddy 的工作流设置就是队长，指挥谁先谁后。"**

### 4.2 复杂文档模板配置：让输出直接可用

**模板配置三步法**：

步骤 1：准备模板文件
- 在 Word/Excel 中设计好最终输出的格式（表头、字体、颜色、公司 Logo 位置）
- 把需要填充数据的位置用 `{{字段名}}` 标注，如 `{{姓名}}`、`{{日期}}`、`{{汇总金额}}`

步骤 2：在 WorkBuddy 中绑定模板
- 打开 WorkBuddy > 文档处理 > 模板管理 > 导入模板
- 上传刚才的 Word/Excel 文件
- 映射数据源：告诉 WorkBuddy `{{姓名}}` 对应哪个输入文件的哪一列

步骤 3：测试与微调
- 用一份测试数据跑一遍，检查输出格式是否正确
- 常见调整：日期格式（2026/04/27 vs 2026-04-27）、数字精度（保留 2 位小数）、空值处理（显示"暂无"还是留空）

**学生项目常见模板场景**：
- 周报模板：从多个 Excel 汇总数据，自动填入周报 Word
- 信息采集表模板：QClaw 采集的信息自动填入标准表格
- 证书/通知书模板：批量生成带不同姓名的文档

**话术提示**：**"模板的价值在于'一次设计，永久复用'。今天花 10 分钟调好模板，以后每次生成报告都是一键完成。"**
