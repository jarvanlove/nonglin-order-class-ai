## 一、封面与使用说明

### 1.1 这本指导书是做什么的

这本指导书是助教独立辅导学生完成 6 天实训课程所有下午实践任务的**唯一操作手册**。目标是一位从未上过这门课的新助教，在课前 2 小时通读本书后，能够独立处理 90% 以上的现场问题。

### 1.2 助教的"三不原则"

**原则一：不替学生做**

学生遇到问题时，你的第一反应不应该是"我帮你弄"。你可以演示操作步骤，但鼠标和键盘必须回到学生手里。学生亲手完成一次操作，记忆留存率是看你操作的 5 倍以上。

**原则二：不直接给答案**

学生问"这个报错什么意思"，不要直接说"你把第 3 行改成 xxx"。先问："你觉得这个报错在说什么？"引导学生自己读错误信息。如果学生完全看不懂，再解释报错含义，然后让学生自己改。

**原则三：不打击积极性**

非计算机专业的学生问出"基础问题"是正常现象。不要表现出"这你都不知道"的态度。你的表情和语气会直接影响学生接下来 3 天的学习状态。如果学生反复问同一个问题，耐心再讲一遍，同时建议他记下来。

### 1.3 什么时候该介入、什么时候让学生自己解决

| 情况 | 助教动作 | 原因 |
|------|---------|------|
| 学生安装软件卡住超过 5 分钟 | 主动上前询问 | 安装是后续一切的前提，不能耽误 |
| 学生指令写错了，QClaw/WorkBuddy 返回错误 | 让学生自己读报错，尝试改一次 | 培养独立排查能力 |
| 组内争论技术方案超过 10 分钟 | 介入拍板，先选一个跑通 | 避免内耗，先完成再完美 |
| 学生情绪低落、明显焦虑 | 立即安抚，必要时降低任务难度 | 心理状态优先于任务完成度 |
| 网络/工具服务故障影响全班 | 立即启动 Plan B（见第 9 章） | 全班性问题不能等 |
| 学生完成基础任务后无所事事 | 分配进阶任务或去帮助其他组员 | 避免时间浪费 |

|------|--------|---------|---------|

**升级规则**：
- 5 分钟内能解决的问题 → 助教独立处理
- 5-15 分钟的问题 → 助教处理，事后在助教群同步

- 工具服务全班故障 → 立即联系技术支持 + 讲师，同步启动 Plan B

### 2.3 14:30 - 15:30 QClaw 安装与配置（核心辅导时段）

这是 Day 1 下午最关键、问题最集中的 60 分钟。助教需要全场巡视，逐个解决安装问题。

#### 步骤 1：打开浏览器访问官网

**操作指令**：
1. 让学生打开电脑上的浏览器（Chrome、Edge、Firefox、360 浏览器均可）
2. 在地址栏输入 `https://qclaw.qq.com`
3. 按回车键

**预期界面描述（截图占位 1）**：
- 页面顶部：腾讯 QClaw 的 Logo 和导航栏（首页、下载、文档、社区）
- 页面中部：大幅宣传图，主标题为"QClaw - 微信远程办公 AI 助手"
- 显眼位置有一个蓝色或绿色的"立即下载"按钮

**常见问题 1：官网打不开**

- **现象**：浏览器显示"无法访问此网站"或长时间加载
- **排查**：
  1. 让学生先访问 `https://www.qq.com`，确认基础网络正常
  2. 如果 qq.com 能打开但 qclaw.qq.com 打不开 → 学校网络对该域名有限制
  3. **解决方案**：让学生连接手机热点，或使用助教准备的离线安装包
- **发生频率**：高（学校网络常见限制）

**常见问题 2：下载速度极慢**

- **现象**：下载进度条几乎不动，预计时间超过 30 分钟
- **排查**：
  1. 检查教室 WiFi 是否带宽不足（30 人同时下载）
  2. 让学生错峰下载：先让一半人下载，另一半 5 分钟后再开始
  3. **解决方案**：助教用 U 盘拷贝安装包，学生直接从 U 盘复制到电脑
- **发生频率**：高

#### 步骤 2：运行安装程序（Windows）

**操作指令**：
1. 打开浏览器下载列表，找到 `QClaw-Setup-x.x.x.exe`
2. 双击文件名运行
3. 若弹出"用户账户控制"（UAC）提示，点击"是"

**预期界面描述（截图占位 2）**：
- 弹出 QClaw 安装向导窗口
- 窗口标题："QClaw 安装"
- 欢迎页面显示 QClaw Logo
- 底部有"下一步"按钮

**常见问题 3：Windows 提示"已保护你的电脑"（SmartScreen）**

- **现象**：弹出蓝色窗口，显示"Windows 已保护你的电脑"，没有"运行"按钮
- **排查**：这是 Windows Defender 对新软件的拦截
- **解决方案**：
  1. 点击窗口上的"更多信息"文字链接
  2. 下方会出现"仍要运行"按钮
  3. 点击"仍要运行"
- **发生频率**：中（Windows 10/11 常见）

**常见问题 4：安装程序闪退或无响应**

- **现象**：双击安装程序后，窗口一闪而过或一直显示"正在加载"
- **排查**：
  1. 检查磁盘空间：打开"此电脑"，看 C 盘可用空间是否大于 500MB
  2. 关闭杀毒软件：临时退出 360、腾讯电脑管家等
  3. 右键安装程序 → "以管理员身份运行"
- **解决方案**：以上三步逐一尝试
- **发生频率**：中

#### 步骤 3：微信扫码绑定

**操作指令**：
1. QClaw 首次启动后，界面显示二维码
2. 学生打开手机微信 → 右上角"+" → "扫一扫"
3. 对准电脑屏幕上的二维码（距离 15-25 厘米）
4. 手机显示"QClaw 登录确认" → 点击"确认登录"
5. 电脑端自动跳转，显示主界面

**预期界面描述（截图占位 3）**：
- 电脑端：二维码消失，显示欢迎信息，顶部有微信头像和昵称
- 底部有输入框，提示文字"输入任务指令..."
- 左侧导航栏有"新建任务"、"历史记录"、"Skills"、"设置"

**常见问题 5：扫码后无反应**

- **现象**：手机显示"扫描成功"，但电脑端没有跳转
- **排查**：
  1. 检查电脑和手机是否都联网
  2. 点击二维码下方的"刷新"按钮，重新扫码
  3. 检查手机微信版本："我" → "设置" → "关于微信"，建议 8.0.40 以上
- **解决方案**：刷新二维码重新扫，或重启 QClaw 后再扫
- **发生频率**：中

**常见问题 6：绑定成功但发送消息无回复**

- **现象**：输入"你好"后，QClaw 长时间不回复（超过 30 秒）
- **排查**：
  1. 检查网络：能否正常打开 qq.com
  2. 检查 QClaw 设置中的 Token 用量
  3. 重启 QClaw：右键任务栏图标 → 退出 → 重新启动
- **解决方案**：通常是网络波动，重启后重试
- **发生频率**：低

### 2.5 15:45 - 17:30 实战任务：农林领域 AI 应用案例搜集

#### 任务讲解与示范（15:45 - 16:15，讲师演示，助教协助准备）

讲师会用 10-15 分钟演示完整流程。助教在此期间：
1. 确认自己的 QClaw 可以正常演示
2. 帮助最后几个未完成安装的学生
3. 准备回答"我的 QClaw 界面和讲师不一样"的问题（版本差异正常）

#### 学生独立完成任务（16:15 - 17:15，助教全场巡视）

**核心任务步骤（学生操作）**：

**步骤 1：新建任务**
1. 点击左侧导航栏"新建任务"
2. 界面中间区域清空，显示"新任务"

**步骤 2：输入信息搜集指令**
1. 点击底部输入框
2. 输入以下指令（可以复制讲师提供的模板）：

```
请搜集 2023 年至 2026 年期间，人工智能在农业或林业领域的应用案例。
要求：
1. 至少找到 5 个不同场景的案例
2. 每个案例包含：名称、具体应用领域、使用的 AI 技术、取得的效果或数据、信息来源链接
3. 场景尽量多样化
4. 优先选择有具体数据支撑的案例
```

3. 按回车发送
4. 等待 QClaw 执行（30 秒到 3 分钟）

**预期界面描述（截图占位 4）**：
- QClaw 显示"正在处理..."状态
- 可能分步显示"正在搜索相关资料..."、"正在整理信息..."
- 最终显示整理好的结果，通常为文本列表或表格

**步骤 3：检查结果并补充**
1. 阅读 QClaw 返回的结果
2. 检查是否满足标准（案例数量 >= 5、要素齐全、场景多样、有数据支撑）
3. 不满足则输入补充指令，例如：
   ```
   请再补充 2 个林业领域的案例，重点关注森林火灾监测方向。
   ```

**步骤 4：格式化输出**
1. 输入指令：
   ```
   请将以上所有案例整理成 Markdown 表格格式，表头为：案例名称 | 应用领域 | AI 技术 | 效果数据 | 来源链接
   ```
2. 等待 QClaw 输出标准 Markdown 表格

**步骤 5：导出保存**
1. 选中全部结果文本（Ctrl+A）
2. 复制（Ctrl+C）
3. 打开记事本或 VS Code
4. 粘贴（Ctrl+V）
5. 保存为 `Day01-农林AI案例搜集-姓名.md`

**常见问题 7：QClaw 回复"无法完成"或"未找到相关信息"**

- **现象**：QClaw 返回错误提示或空结果
- **排查**：
  1. 检查指令是否太模糊："搜一下农业 AI" → 改为"搜集 2023-2026 年 AI 在病虫害识别领域的应用案例"
  2. 分步执行：先让搜案例，再让整理格式
  3. 检查 Skills 是否启用：左侧"Skills" → 确认"搜索"已启用
- **解决方案**：优化指令表述，分步执行
- **发生频率**：中

**常见问题 8：QClaw 输出格式每次都不一样**

- **现象**：同样指令，两次返回的格式不同
- **排查**：AI 输出的固有不确定性
- **解决方案**：
  1. 在指令中明确指定输出格式："请严格按照以下 JSON 格式输出..."
  2. 给出示例：在指令中贴一个期望的输出样例
- **发生频率**：中

**常见问题 9：学生不知道搜集什么方向**

- **现象**：学生盯着屏幕发呆，不知道输入什么关键词
- **解决方案**：提供参考方向（见下方"参考案例方向"）
- **发生频率**：高（非计算机专业学生常见）

**参考案例方向（给学生看的提示卡）**：
1. 病虫害识别：基于图像识别的作物病虫害自动诊断
2. 产量预测：基于气象数据和机器学习的水稻/小麦产量预测
3. 智慧灌溉：基于土壤传感器和 AI 的精准灌溉决策
4. 林业遥感：基于卫星遥感和深度学习的森林覆盖变化监测
5. 农产品分级：基于计算机视觉的水果/蔬菜自动分级分拣
6. 畜牧监测：基于物联网和 AI 的畜禽健康监测与疫病预警
7. 无人机植保：AI 路径规划的无人机精准施药

#### 时间提醒：16:45

此时应该完成的状态：
- 至少 70% 的学生已完成信息搜集，正在整理格式
- 剩余 30% 的学生至少已完成案例搜集，正在格式化
- 如果超过 10 人还在搜集阶段，提醒他们先用 3 个案例完成基础任务，再补充

### 3.2 14:00 - 14:30 WorkBuddy 安装与界面导览

#### 步骤 1：下载安装包

**操作指令**：
1. 打开浏览器，访问 `https://workbuddycn.com`

**常见问题 3：macOS 提示"无法打开 WorkBuddy"**

**操作指令**：
1. 确认主界面四个区域：顶部导航栏、左侧边栏、中间对话区、底部输入区

3. 弹出三个模式选项 → 选择"Craft 模式"（锤子图标）
4. 弹出权限说明弹窗 → 勾选"我已了解风险" → 点击"确认开启"

**预期界面描述（截图占位 5）**：
- 对话区顶部显示 Craft 模式图标（锤子）
- 底部输入区显示回形针图标（上传文件）可用

### 3.3 14:30 - 15:30 核心功能分步练习

#### 练习 1：本地文件处理（20 分钟）

**操作指令**：

**常见问题 5：上传文件夹按钮灰色不可用**

- **现象**：点击回形针后，"上传文件夹"选项灰色
- **原因**：未开启 Craft 模式
- **解决方案**：切换到 Craft 模式
- **发生频率**：高（学生经常忘记切换）

**常见问题 6：提示"无法访问该路径"**

- **现象**：选择文件夹后报错
- **原因**：文件夹路径含中文特殊字符或权限不足
- **解决方案**：将文件夹复制到桌面再上传
- **发生频率**：中

#### 练习 2：Excel 数据清洗（20 分钟）

**操作指令**：
1. 点击回形针图标 → 选择"上传文件"
2. 选择 `day02-practice-data/sales_data.xlsx`
3. 输入指令查看数据概况：
   ```
   请读取这个 Excel 文件，告诉我它有多少行、多少列，列名分别是什么。
   ```
4. 输入数据清洗指令：
   ```
   请帮我清洗这份数据：
   1. 删除完全重复的行
   2. 填充空值：数值列用0填充，文本列用"未填写"填充
   3. 统一日期格式为 YYYY-MM-DD
   4. 金额列统一为纯数字，去掉人民币符号和千分位逗号
   5. 将清洗后的数据保存到 output/cleaned_sales.xlsx
   ```
5. WorkBuddy 显示执行计划 → 点击"确认执行"
6. 打开 `output/cleaned_sales.xlsx` 验证结果

**常见问题 7：提示"无法解析 Excel 文件"**

- **现象**：上传后 WorkBuddy 报错
- **原因**：文件是 .xls 老格式或已损坏
- **解决方案**：用 Excel 打开后另存为 .xlsx 格式再上传
- **发生频率**：中

**常见问题 8：清洗后数据行数没变**

- **现象**：重复行未被删除
- **原因**：WorkBuddy 未识别出重复行
- **解决方案**：在指令中明确指定"根据 A 列和 B 列联合判断是否重复"
- **发生频率**：中

**常见问题 9：金额列仍含符号**

- **现象**：清洗后金额列还有人民币符号
- **原因**：符号为全角字符（如￥）
- **解决方案**：指令中写"去掉所有非数字字符，保留小数点"
- **发生频率**：中

#### 练习 3：批量重命名与归档（20 分钟）

**操作指令**：
1. 确认 Craft 模式已开启
2. 上传 `day02-practice-data/raw_files/`
3. 输入批量重命名指令：
   ```
   请将这个文件夹中的所有图片文件重命名为 "photo_001.jpg", "photo_002.jpg" 的格式，
   文本文件重命名为 "note_001.txt", "note_002.txt" 的格式。
   保持原有扩展名不变，编号从 001 开始按字母顺序排列。
   ```
4. WorkBuddy 列出预览 → 检查无误后点击"确认执行"
5. 输入归档指令：
   ```
   请在这个文件夹下创建两个子文件夹："images" 和 "documents"，
   然后将所有图片文件移动到 "images"，所有文本文件移动到 "documents"。
   ```
6. 确认预览后点击"确认执行"
7. 打开文件管理器验证结果

**预期输出样例**：
```
raw_files/
├── images/
│   ├── photo_001.jpg
│   ├── photo_002.jpg
│   └── photo_003.jpg
└── documents/
    ├── note_001.txt
    └── note_002.txt
```

**常见问题 10：重命名后文件丢失**

- **现象**：重命名后文件夹里文件少了
- **原因**：文件名冲突导致覆盖
- **解决方案**：在指令中补充"如遇冲突，在文件名后加 _1"
- **发生频率**：低

### 3.5 15:45 - 17:30 综合任务：周报自动化生成

#### Step 1：数据提取（30 分钟）

**操作指令**：
1. 上传所有源文件（按住 Ctrl/Command 多选）：
   - `sales_data.xlsx`
   - `meeting_notes_1.docx`
   - `meeting_notes_2.docx`
   - `project_status.xlsx`
2. 依次发送三个提取指令（见下方）
3. 记录 WorkBuddy 返回的结果

**提取销售数据指令**：
```
请读取 sales_data.xlsx，提取以下信息并用 bullet points 列出：
- 本周总销售额
- 销售额最高的三个产品及其金额
- 与上周相比的增长率（假设上周总销售额为 45000）
```

**提取会议纪要指令**：
```
请读取两个 Word 文件，分别提取：
1. 会议决议（Decisions）
2. 待办事项（Action Items），标注负责人和截止日期
3. 识别的风险或问题
```

**提取项目进度指令**：
```
请读取 project_status.xlsx，列出：
- 各项目名称及完成百分比
- 状态为"阻塞"的项目及阻塞原因
- 预计本周内完成的项目
```

**常见问题 11：上传多个文件后 AI 只识别了其中一个**

- **现象**：发送指令后，WorkBuddy 只处理了第一个文件
- **原因**：文件上传有延迟，AI 在全部上传完成前已处理
- **解决方案**：等待所有文件显示"已上传"后再发送指令
- **发生频率**：高

**常见问题 12：提取的信息与文件内容不符**

- **现象**：WorkBuddy 返回的内容和文件实际内容对不上
- **原因**：AI 混淆了多个文件的内容
- **解决方案**：在指令中明确指定文件名，如"读取 sales_data.xlsx"
- **发生频率**：中

#### Step 2：生成图表数据（20 分钟）

**操作指令**：
1. 发送销售图表数据指令：
   ```
   基于 sales_data.xlsx 的数据，生成一个适合制作柱状图的表格数据：
   列：产品名称 | 本周销售额 | 上周销售额
   按销售额从高到低排列，输出为 Markdown 表格格式。
   ```
2. 发送项目进度图表数据指令：
   ```
   基于 project_status.xlsx，生成适合制作甘特图或进度条图的数据：
   列：项目名称 | 计划完成日期 | 实际完成百分比 | 状态
   输出为 Markdown 表格。
   ```
3. 发送图表插入说明指令：
   ```
   请为上述两组数据分别写一段"图表插入说明"，告诉我：
   1. 推荐用什么类型的图表展示
   2. 图表标题建议
   3. 关键数据点标注建议
   ```

#### Step 3：生成格式化周报（40 分钟）

**操作指令**：
1. 发送周报生成指令：
   ```
   请基于我们刚才提取的所有信息，生成一份周报正文，要求：
   - 标题：第 X 周工作周报（2026年4月第4周）
   - 结构：
     一、本周工作概述（2-3句话总结）
     二、销售数据（含关键数字）
     三、会议决议与待办
     四、项目进度
     五、下周计划（基于待办事项和项目进度合理推测）
   - 语言风格：正式、简洁、数据驱动
   ```
2. 发送保存指令：
   ```
   请将上述周报内容保存为 Word 文档，路径：day02-practice-data/output/weekly_report.docx
   格式要求：
   - 标题用"标题 1"样式
   - 各章节用"标题 2"样式
   - 正文用"正文"样式
   - 关键数据加粗
   ```
3. 打开 `output/weekly_report.docx` 验证

**常见问题 13：生成的周报内容空洞**

- **现象**：周报内容泛泛而谈，没有具体数字
- **原因**：指令不够具体
- **解决方案**：在指令中明确要求"包含具体数字"、"引用原文"
- **发生频率**：中

**常见问题 14：Word 文档保存后打不开**

- **现象**：双击 docx 文件报错"文件已损坏"
- **原因**：格式指令过于复杂导致生成错误
- **解决方案**：简化格式要求，先保存纯文本再手动排版
- **发生频率**：低

**常见问题 15：文件路径错误导致保存失败**

- **现象**：WorkBuddy 提示"路径不存在"
- **原因**：output 文件夹未创建，或路径含特殊字符
- **解决方案**：确保 output 文件夹已存在，使用英文路径
- **发生频率**：中

### 4.5 14:50 - 15:30 需求拆解

#### 三步法实操辅导

**第一步：用户故事（10 分钟）**

**操作指令**：
1. 每组在白板/纸上写出至少 3 个用户故事
2. 格式：
   ```
   作为一个 [角色]，
   我想要 [功能]，
   以便 [价值/目的]。
   ```

**助教巡视要点**：
- 检查用户故事是否从"用户视角"出发，而不是"我们要做什么功能"
- 纠正典型错误："我们要做一个搜索功能" → 改为"作为一个学生，我想要搜索植物信息，以便快速找到某种植物的资料"

**操作指令**：
1. Trae 模块能读取 WorkBuddy 处理后的数据文件

3. 界面交互功能正常（筛选、搜索等）
4. 错误处理：数据文件缺失时显示友好提示

**Trae 读取本地 JSON 代码模板**：
```javascript
fetch('./data/products.json')
  .then(response => {
    if (!response.ok) {
      throw new Error('数据文件加载失败');
    }
    return response.json();
  })
  .then(data => {
    console.log('加载成功，共', data.length, '条数据');
    renderProducts(data);
  })
  .catch(error => {
    console.error(error);
    document.getElementById('error-msg').textContent = '数据加载失败，请检查数据文件是否存在';
  });

## 九、紧急应对方案（Plan B）

### 9.1 QClaw 服务不稳定时的替代方案

**触发条件**：全班超过 50% 的学生无法正常使用 QClaw（持续 15 分钟以上）

**替代方案 A：本地脚本模拟**
1. 助教提前准备 Python 脚本 `mock_qclaw.py`
2. 脚本功能：读取本地关键词文件，模拟生成 JSON 输出
3. 学生运行脚本代替 QClaw 采集

```python
# mock_qclaw.py 示例
import json
import random

keywords = ["智慧农业", "农产品价格", "农业政策"]
results = []
for kw in keywords:
    results.append({
        "title": f"{kw}相关新闻",
        "source": "模拟来源",
        "date": "2026-04-27",
        "summary": f"关于{kw}的模拟摘要",
        "url": "https://example.com",
        "category": "模拟"
    })

with open("qclaw-output.json", "w", encoding="utf-8") as f:
    json.dump(results, f, ensure_ascii=False, indent=2)
```

**替代方案 B：简化任务**
1. 取消实时采集，改用助教提前准备好的数据文件
2. 学生任务改为"分析已有数据"而非"采集新数据"
3. 重点保留"数据处理"和"工具使用"的训练价值

**替代方案 C：网页版替代**
1. 如果 QClaw 客户端不可用，尝试网页版（如有）
2. 或改用其他国产 AI 工具（如文心一言、通义千问）完成信息搜集

### 9.2 WorkBuddy 无法安装时的替代方案

**触发条件**：超过 30% 的学生无法完成 WorkBuddy 安装

**替代方案 A：本地 Python 脚本**
1. 用 Python + pandas 完成数据清洗
2. 用 Python + openpyxl 生成 Excel 报告
3. 助教提供脚本模板，学生修改参数即可

**替代方案 B：Excel 内置功能**
1. 用 Excel 的"数据"→"删除重复项"完成去重
2. 用 Excel 公式完成格式统一
3. 用 Excel 模板生成报告

**替代方案 C：WPS AI**
1. 如果 WorkBuddy 不可用，尝试 WPS Office 的 AI 功能
2. WPS AI 同样支持文档处理和数据分析

### 9.3 Trae Builder 模式崩溃时的回退方案

**触发条件**：Trae Builder 模式频繁崩溃或无法生成代码

**回退方案 A：Chat 模式手动开发**
1. 改用 Trae 的 Chat 模式
2. 学生分步提问："帮我写 HTML 骨架"→"帮我写 CSS 样式"→"帮我写 JS 逻辑"
3. 手动复制代码到文件中

**回退方案 B：简化项目需求**
1. 将"动态网页"降级为"静态网页"
2. 将"数据可视化"降级为"表格展示"
3. 确保基础功能可用，放弃进阶功能

**回退方案 C：使用在线代码编辑器**
1. 如果 Trae 完全不可用，使用 CodePen、JSFiddle 等在线编辑器
2. 或直接使用 VS Code + 国产 AI 插件（如通义灵码）

### 9.4 学校网络突然中断时的应对方案

**触发条件**：教室网络完全中断，无法访问任何网站

**应对方案**：
1. **立即切换为纯本地模式**：所有操作不需要网络
2. **QClaw 替代**：使用提前准备的本地数据文件，任务改为"分析本地数据"
3. **WorkBuddy 替代**：如果 WorkBuddy 已安装且支持本地模式，继续使用；否则用本地脚本
4. **Trae 替代**：Trae 本地 IDE 模式不依赖网络，可继续使用
5. **讲师转为讲授**：利用断网时间讲授理论知识、案例分析、职业指导

**课前预防措施**：
1. 所有安装包、数据文件、脚本模板提前拷贝到 U 盘
2. 准备"断网版课程大纲"（纯讲授内容）
3. 确认教室是否有备用网络（如手机热点覆盖）

### 9.5 学生电脑配置过低无法运行 Trae 时的兜底方案

**触发条件**：学生电脑内存 < 4GB 或 CPU 过旧，Trae 卡顿/无法启动

**兜底方案 A：云端开发环境**
1. 使用 GitHub Codespaces（如果网络允许）
2. 使用腾讯云 Cloud Studio 等国内云端 IDE
3. 学生在本机编辑简单 HTML，上传到云端运行

**兜底方案 B：降级为纯文本编辑**
1. 使用记事本/VS Code 编辑 HTML/CSS/JS
2. 用浏览器直接打开 HTML 文件预览
3. 放弃 Trae 的 AI 辅助功能，改为手动编写

**兜底方案 C：组内共享电脑**
1. 让配置好的学生电脑作为"开发机"
2. 配置低的学生负责需求文档、测试、资料搜集
3. 通过 Git 或 U 盘同步代码

**兜底方案 D：助教电脑协助**
1. 极端情况下，允许学生在助教电脑上完成关键操作
2. 助教演示操作，学生记录步骤
3. 课后将代码文件拷贝给学生

## 十、学生求助决策树

### 10.1 核心原则

```
学生举手/喊助教
    │
    ▼
判断问题类型
    │
    ├─→ 安装/环境问题（QClaw/WorkBuddy/Trae 装不上、打不开）
    │     └─→ 立即介入，这是后续一切的前提
    │
    ├─→ 操作步骤问题（不知道点哪里、不知道输入什么）
    │     └─→ 口头提示，让学生自己操作
    │
    ├─→ 报错/异常问题（软件报错、结果不对）
    │     └─→ 让学生先读报错信息，尝试自己解决
    │           └─→ 5 分钟后仍无法解决 → 助教演示
    │
    ├─→ 设计/决策问题（"这个功能要不要做""用哪个方案"）
    │     └─→ 推给组长或讲师，助教不做设计决策
    │
    └─→ 情绪/心理问题（焦虑、沮丧、想放弃）
          └─→ 立即安抚，必要时降低任务难度，联系讲师
```

### 10.2 时间分级处理标准

| 预计解决时间 | 处理方式 | 示例 |
|-------------|---------|------|
| 1-2 分钟 | 口头提示，学生自己操作 | "你点击左侧的'新建任务'按钮" |
| 3-5 分钟 | 站在旁边指导，学生操作 | 指导学生排查网络问题 |
| 5-15 分钟 | 助教演示一遍，学生跟着做 | 演示 WorkBuddy 的 Craft 模式配置 |
| 15-30 分钟 | 助教协助完成，事后讲解 | 帮学生搭建项目目录结构 |
| 超过 30 分钟 | 推给讲师，或启动 Plan B | 工具服务故障、复杂技术问题 |

### 10.3 不同学生类型的应对策略

| 学生类型 | 特征 | 应对策略 |
|---------|------|---------|
| 积极主动型 | 提前完成任务，问进阶问题 | 分配进阶任务，鼓励帮助其他同学 |
| 按部就班型 | 跟着步骤做，不求快但求稳 | 给予明确步骤，及时肯定进度 |
| 依赖求助型 | 每一步都要问 | 坚持"三不原则"，引导独立解决 |
| 沉默挣扎型 | 不说话，但明显卡住 | 主动上前询问，提供温和帮助 |
| 焦虑急躁型 | 频繁叹气、敲键盘很重 | 安抚情绪，降低任务难度，分解步骤 |
| 组内边缘型 | 不参与讨论，被分配简单任务 | 主动询问意见，分配有成就感的任务 |

### 10.4 情绪崩溃处理流程

**识别信号**：
- 学生突然停止操作，盯着屏幕发呆
- 学生叹气频率明显增加
- 学生说"我不行了""我做不完"
- 学生眼眶发红或流泪

**处理步骤**：
1. **立即到场**：走到学生身边，蹲下或坐下（不要站着俯视）
2. **温和询问**："我看你好像遇到了困难，能跟我说说吗？"
3. **倾听不打断**：让学生说完，不要急于给解决方案
4. **降低难度**："我们先不追求完美，先把基础任务完成就好"
5. **分解任务**："接下来你只需要做这一件事..."
6. **陪伴完成**：站在旁边陪着完成第一步，建立信心
7. **事后跟进**：15 分钟后回来检查状态
8. **必要时联系讲师**：如果情绪持续低落，联系讲师处理

**绝对禁止**：
- "这么简单你都不会？"
- "别人都能做完"
- "那你晚上加班做"
- 在学生面前表现出不耐烦
