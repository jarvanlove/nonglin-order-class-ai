## Day 1 — QClaw 案例（信息收集与自动化）

### 案例 1-1 【基础】农业 AI 应用案例收集

**目标**：让学生熟悉 QClaw 基本对话和网页搜索能力。

**完整提示词（可复制给学生）**：

```
请帮我搜索并整理 3 个国内农业领域应用 AI 的真实案例。要求：
1. 每个案例包含：公司名称、应用场景、使用的 AI 技术、实际效果（数据）
2. 优先选择 2024-2025 年的案例
3. 输出格式为表格
4. 如果找不到具体数据，标注"数据未公开"
```

**预期输出**：

| 公司名称 | 应用场景 | AI 技术 | 实际效果 |
|---------|---------|---------|----------|
| 极飞科技 | 无人机植保 | 计算机视觉 + 路径规划 | 作业效率提升 30%，农药使用减少 20% |
| 佳格天地 | 卫星遥感监测 | 遥感影像识别 + 时序分析 | 作物识别准确率 95% 以上 |
| 丰农控股 | 农技知识问答 | 大语言模型 + 知识图谱 | 覆盖 100+ 种作物，问答准确率 90% |

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 案例过于陈旧 | 给出 2018-2019 年的案例 | 提醒学生明确要求"2024-2025 年"，QClaw 会优先检索最新信息 |
| 输出格式混乱 | 用段落而非表格输出 | 强调"输出格式为表格"这一指令的重要性，格式指令要放在提示词末尾 |
| 虚构数据 | 编造不存在的公司名称或数据 | 要求学生交叉验证：用 QClaw 搜索案例中的公司名，确认是否真实存在 |

**讲师应答脚本**：

- **学生问"搜不到案例怎么办"**："先尝试用更宽泛的关键词，比如'智慧农业'、'AI 种植'。如果还是搜不到，可能是网络问题，请举手助教协助。"
- **学生问"数据不准怎么办"**："AI 搜索的结果需要人工验证。至少做一步验证：点击 QClaw 提供的来源链接，确认信息出处。"

### 案例 1-2 【进阶】定时天气监测任务配置

**目标**：让学生掌握 QClaw 的定时任务和插件调用能力。

**完整提示词（可复制给学生）**：

```
请帮我创建一个定时任务：
1. 每天早上 8:00 查询"北京市海淀区"当天的天气
2. 如果气温低于 5 度或高于 35 度，或预报有雨雪，请发送微信消息提醒我注意农事防护
3. 消息格式："【天气提醒】今日北京海淀气温 X°C，天气状况 XX，建议：XX"
4. 任务名称：海淀农事天气监测
```

**预期输出**：

QClaw 会引导用户完成以下步骤：
1. 确认定时任务时间（cron 表达式：0 8 * * *）
2. 确认天气查询插件已启用
3. 确认微信消息推送已绑定
4. 生成任务摘要卡片，显示下次执行时间

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 城市名称写错 | "北京海淀"而非"北京市海淀区"导致定位失败 | 提醒学生使用标准行政区划名称，可在 QClaw 中先测试一次天气查询确认定位 |
| 未绑定微信 | 任务创建成功但无法收到消息 | 强调必须先完成微信绑定才能接收推送，在"设置-账号绑定"中检查 |
| 条件设置过宽/过严 | 全部天气都提醒或从不提醒 | 引导学生理解"触发条件"的含义，建议先用极端值测试（如设置 30 度触发，确认今天会收到） |

**讲师应答脚本**：

- **学生问"定时任务没执行怎么办"**："先检查三点：① 任务状态是否为'启用'；② QClaw 是否在线；③ 微信是否屏蔽了 QClaw 消息。可在任务列表中手动点击'立即执行'测试。"
- **学生问"能不能监测多个地区"**："可以，复制任务并修改地区名称。QClaw 支持为不同任务设置不同触发条件。"

### 案例 1-3 【挑战】农业舆情监测 Agent 搭建

**目标**：让学生综合运用搜索、定时任务、微信推送，搭建一个简易的"农业舆情监测 Agent"。

**完整提示词（可复制给学生）**：

```
请帮我搭建一个农业舆情监测助手，需求如下：
1. 每小时搜索一次"农产品价格 波动"、"农业灾害 预警"、"农药 召回"三个关键词的最新新闻
2. 发现新闻后，用一句话总结核心内容
3. 如果新闻涉及"粮食安全"、"大规模灾害"、"食品安全问题"，立即微信推送并标注【紧急】
4. 普通新闻每天 18:00 汇总推送一次
5. 任务名称：农业舆情监测

请分步骤告诉我如何配置，并生成每个步骤的提示词。
```

**预期输出**：

QClaw 会输出分步配置指南：
1. **创建搜索任务 A（紧急监测）**：每 1 小时搜索关键词，设置过滤条件（包含"粮食安全"等），开启即时微信推送
2. **创建搜索任务 B（日常汇总）**：每 1 小时搜索关键词，不包含紧急关键词的新闻存入临时列表
3. **创建定时任务 C（日报推送）**：每天 18:00 汇总任务 B 的临时列表，生成摘要推送
4. **测试验证**：提供测试关键词（如"粮食安全 预警"），确认紧急推送通道正常

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 任务冲突 | 紧急新闻同时触发任务 A 和 B，导致重复推送 | 提醒学生在任务 B 中设置排除条件（不包含任务 A 的关键词），或接受少量重复，优先保证不漏报 |
| 搜索频率过高 | 每小时搜索导致信息过载 | 引导学生根据实际需求调整频率，舆情监测通常"每小时"或"每 6 小时"即可，不需要实时 |
| 关键词过于宽泛 | "农业"一词导致大量无关新闻 | 建议关键词组合使用（如"农产品价格 + 波动"），并设置排除词（如"股票"、"期货"） |

**讲师应答脚本**：

- **学生问"这个 Agent 能商用吗"**："这是教学演示版本，商用需考虑：① 新闻版权；② 信息准确性；③ 推送频率合规。实际商用建议使用正规新闻 API（如百度舆情、清博大数据）。"
- **学生问"能不能监测社交媒体"**："QClaw 的网页搜索主要覆盖公开网页，社交媒体（微博、抖音）需特定插件或 API。这是进阶话题，课后可探讨。"

## Day 2 — WorkBuddy 案例（本地文件处理与自动化）

### 案例 2-1 【基础】Excel 数据清洗与统计

**目标**：让学生掌握 WorkBuddy 的 Excel 读取、清洗、统计能力。

**完整提示词（可复制给学生）**：

```
请帮我处理桌面上的 "agriculture_data.xlsx" 文件，完成以下操作：
1. 读取文件，查看前 10 行数据
2. 删除所有"产量"列为空的行
3. 将"日期"列统一格式为 YYYY-MM-DD
4. 按"作物类型"分组，统计每类作物的平均产量和最高产量
5. 将结果保存为新文件 "agriculture_summary.xlsx"
6. 生成一段文字摘要，说明数据概况
```

**配套数据文件（讲师准备）**：

`agriculture_data.xlsx` 应包含以下列：日期、地区、作物类型、产量(吨)、种植面积(亩)、备注。其中故意设置：
- 3-5 行产量为空
- 日期格式不统一（2024/1/15、2024-02-20、2024.03.10）
- 1-2 行备注含异常字符

**预期输出**：

1. WorkBuddy 读取文件并展示前 10 行预览
2. 清洗后数据行数（如从 100 行减少到 95 行）
3. 日期格式统一后的样本
4. 分组统计表格：

| 作物类型 | 平均产量(吨) | 最高产量(吨) | 记录数 |
|---------|-------------|-------------|--------|
| 小麦 | 520 | 890 | 32 |
| 玉米 | 680 | 1200 | 28 |
| 水稻 | 750 | 1350 | 35 |

5. 新文件保存成功提示
6. 文字摘要示例："本次数据涵盖 95 条记录，涉及 3 类作物。水稻平均产量最高（750 吨），玉米产量波动最大（最高 1200 吨）。"

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 文件路径错误 | WorkBuddy 找不到文件 | 提醒学生确认文件确实在桌面，或提供完整路径（Windows：C:\Users\XXX\Desktop\...；macOS：/Users/XXX/Desktop/...） |
| 日期格式转换失败 | 部分日期仍为原格式 | 解释 Excel 中日期可能是"文本型"，需先转换为日期型再格式化。WorkBuddy 会自动处理大部分情况，如遇异常可手动指定格式 |
| 分组统计结果为空 | 作物类型列有拼写不一致（如"小麦"和"小麦 "） | 引导学生观察原始数据，发现隐藏空格或换行符问题。建议在清洗步骤增加"去除首尾空格"操作 |

**讲师应答脚本**：

- **学生问"WorkBuddy 能处理多大的文件"**："10 万行以内的 Excel 通常没问题。如果文件特别大（百万行以上），建议先用 WorkBuddy 拆分成小文件处理。"
- **学生问"清洗规则能不能保存复用"**："可以。WorkBuddy 支持将操作步骤保存为'模板'，下次直接套用。点击'保存模板'并命名即可。"

### 案例 2-2 【进阶】批量文件重命名与整理

**目标**：让学生掌握 WorkBuddy 的文件系统操作和正则表达式基础。

**完整提示词（可复制给学生）**：

```
请帮我整理桌面上的 "raw_photos" 文件夹，完成以下操作：
1. 读取文件夹内所有图片文件（.jpg, .png, .jpeg）
2. 按文件名中的日期信息（如 IMG_20240115_001.jpg）提取日期
3. 创建以日期命名的子文件夹（如 2024-01-15），将对应图片移入
4. 如果文件名不含日期，统一重命名为 "未知日期_序号.扩展名"
5. 生成整理报告：原文件数、整理后文件夹数、未识别日期文件数
```

**配套测试文件夹（讲师准备）**：

`raw_photos` 文件夹内含 15-20 张图片，文件名设计：
- 10 张含日期：IMG_20240115_001.jpg、DSC_20240220.png、20240310_farm.jpg
- 5 张不含日期：temp1.jpg、image.png、backup.jpg
- 2 张非图片：readme.txt、data.csv（测试过滤逻辑）

**预期输出**：

1. 识别到 15 个图片文件（过滤掉 2 个非图片）
2. 创建 3 个日期子文件夹：2024-01-15、2024-02-20、2024-03-10
3. 移动 10 张含日期图片到对应文件夹
4. 重命名 5 张无日期图片为：未知日期_01.jpg、未知日期_02.png ...
5. 整理报告：
   ```
   原文件数：17
   识别图片：15
   创建文件夹：3
   已整理：10
   未识别日期：5
   完成时间：2024-XX-XX XX:XX
   ```

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 日期格式识别不全 | 20240310_farm.jpg 未被识别 | 解释 WorkBuddy 的日期识别逻辑（优先匹配标准格式），建议对非标准文件名先用"批量重命名"统一格式 |
| 误删文件 | 整理后原文件夹空了，学生担心丢失 | 强调 WorkBuddy 默认是"移动"而非"删除"，且操作前会自动备份。可在"回收站"或 WorkBuddy 的"操作历史"中恢复 |
| 子文件夹创建失败 | 提示权限不足 | 检查文件夹是否被其他程序占用，或路径是否过长。建议将 raw_photos 复制到桌面再操作 |

**讲师应答脚本**：

- **学生问"能不能按其他维度整理"**："可以。提示词中的'日期'可以替换为'地点'、'作物类型'等任何能从文件名或 EXIF 信息中提取的维度。"
- **学生问"整理错了怎么撤销"**："WorkBuddy 有'操作历史'面板，选中本次操作点击'撤销'即可。如果已经关闭了 WorkBuddy，去系统回收站找回。"

### 案例 2-3 【挑战】周报自动化生成系统

**目标**：让学生综合运用 Excel 处理、模板填充、文件操作，搭建一个"周报自动生成"工作流。

**完整提示词（可复制给学生）**：

```
请帮我搭建一个周报自动生成系统，需求如下：

数据源：
- 桌面上有 5 个 Excel 文件，分别是 monday.xlsx 到 friday.xlsx
- 每个文件包含列：任务名称、完成状态（已完成/进行中/未开始）、耗时(小时)、备注

输出要求：
1. 读取 5 个文件，汇总所有任务
2. 统计：总任务数、已完成数、进行中数、总耗时
3. 按"完成状态"分组列出任务清单
4. 生成一份 Word 文档，使用以下模板格式：

   === 周报模板 ===
   汇报人：[姓名]
   周期：第 X 周（[起始日期] - [结束日期]）

   一、本周概览
   总任务：X 项 | 已完成：X 项 | 进行中：X 项 | 总耗时：X 小时

   二、已完成任务
   1. [任务名] - 耗时 X 小时 - [备注]
   ...

   三、进行中任务
   1. [任务名] - 已耗时 X 小时 - [备注]
   ...

   四、下周计划
   （根据"进行中"和"未开始"任务自动生成）
   ...

5. 将生成的 Word 文档保存为 "周报_第X周_姓名.docx"
6. 生成完成后，用一句话总结本周工作效率

请分步骤执行，每步完成后向我确认。
```

**配套数据文件（讲师准备）**：

5 个 Excel 文件，每个含 3-6 条任务记录，设计：
- 部分任务跨天出现（如"数据分析"在 monday 和 tuesday 都有记录，需去重）
- 部分备注含多行文本（测试 Word 排版）
- 1 个文件含"耗时"为空（测试异常处理）

**预期输出**：

1. 步骤 1：读取并展示 5 个文件的概览（行数、列名）
2. 步骤 2：汇总统计（总任务 23 项，已完成 15 项，进行中 6 项，未开始 2 项，总耗时 48 小时）
3. 步骤 3：去重后的任务清单（跨天任务合并耗时）
4. 步骤 4：生成的 Word 文档预览（前 20 行）
5. 步骤 5：保存成功提示，提供文件路径
6. 步骤 6：效率总结（如"本周任务完成率 65%，平均每个任务耗时 2.1 小时，效率良好"）

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 跨天任务重复统计 | "数据分析"在周报中出现两次 | 引导学生理解"去重"逻辑：相同任务名合并耗时，状态取最新一天的状态 |
| Word 排版混乱 | 备注中的换行符导致格式错乱 | 解释 WorkBuddy 会自动处理大部分格式，如遇异常可在模板中预留富文本区域 |
| 耗时为空导致统计错误 | 总耗时计算报错或结果异常 | 提醒学生在提示词中增加异常处理说明（如"耗时为空时按 0 计算"），或手动补全数据 |

**讲师应答脚本**：

- **学生问"能不能改成月报"**："可以。将 5 个日报文件替换为 4 个周报文件，模板中的'本周'改为'本月'即可。WorkBuddy 的模板是通用的。"
- **学生问"能不能自动发送邮件"**："WorkBuddy 支持调用邮件客户端或 SMTP。这是扩展任务，完成基础版本后可尝试添加'发送给主管'步骤。"

## Day 3 — Trae 案例（AI 辅助编程）

### 案例 3-1 【基础】农产品价格展示页面

**目标**：让学生熟悉 Trae Builder 模式，从自然语言生成可运行代码。

**完整提示词（可复制给学生，在 Trae Builder 中使用）**：

```
请帮我创建一个农产品价格展示网页，要求：
1. 使用 HTML + CSS + JavaScript，单文件即可
2. 页面标题："今日农产品价格"
3. 展示 5 种农产品的价格卡片：苹果、香蕉、西红柿、黄瓜、土豆
4. 每种农产品包含：名称、今日价格（元/斤）、涨跌幅度（用红色表示涨，绿色表示跌）、产地
5. 使用模拟数据（不需要真实 API）
6. 页面风格清新自然，有农业气息（绿色系）
7. 响应式布局，在手机和电脑上都能正常显示
8. 添加简单的交互动画：鼠标悬停时卡片微微上浮

请直接生成完整代码，我需要在浏览器中打开就能运行。
```

**预期输出**：

Trae Builder 生成一个完整的 `index.html` 文件（约 150-200 行），包含：
- HTML 结构：header + 5 个 card 容器
- CSS：绿色系配色（#4CAF50 等）、flex/grid 响应式布局、hover 动画（transform: translateY + box-shadow）
- JavaScript：模拟数据数组、动态渲染函数、涨跌颜色判断逻辑

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| 代码生成后不会运行 | 学生不知道在哪打开 | 强调 Trae 右上角的"在浏览器中打开"按钮，或教学生在资源管理器中找到文件双击打开 |
| 修改需求后代码被覆盖 | 学生在 Builder 中输入新需求，原代码丢失 | 提醒学生 Builder 每次会重新生成，如需迭代修改请使用 Chat 模式选中特定代码块修改 |
| 样式不满意 | 学生觉得"不够好看" | 引导学生用 Chat 模式精确修改：选中 CSS 部分，输入"把主色调改成橙色系，更有丰收的感觉" |

**讲师应答脚本**：

- **学生问"怎么添加更多农产品"**："在 JavaScript 的数据数组中添加新对象即可。格式：{name: '西瓜', price: 2.5, change: +0.3, origin: '新疆'}。"
- **学生问"怎么接入真实价格数据"**："基础版本用模拟数据。进阶做法是调用免费 API（如农业农村部数据接口），需要处理跨域和异步加载，这是明天的内容。"

### 案例 3-2 【进阶】天气查询小应用

**目标**：让学生掌握 Trae 中的 API 调用、异步处理、错误处理。

**完整提示词（可复制给学生）**：

```
请帮我创建一个天气查询网页应用，要求：
1. 使用 HTML + CSS + JavaScript，单文件
2. 页面有一个输入框，让用户输入城市名（如"北京"、"上海"）
3. 点击"查询"按钮后，调用和风天气 API（QWeather）获取实时天气
4. 显示：城市名、当前温度、天气状况（晴/雨/多云等）、风向风力、空气湿度
5. 如果城市不存在或网络错误，显示友好的错误提示（如"未找到该城市，请检查拼写"）
6. 页面风格简洁清爽，天气图标用 emoji 或简单 CSS 图形表示
7. 添加加载动画，查询时显示"正在获取天气..."

API 信息：
- 使用 QWeather 免费版 API：https://dev.qweather.com/
- 需要申请免费 API Key（在网站上注册即可获取）
- 当前天气接口文档：https://dev.qweather.com/docs/api/weather/weather-now/

请生成完整代码，并在代码注释中标注：① 哪里需要填入 API Key；② 主要函数的作用。
```

**预期输出**：

Trae 生成完整的 `weather.html`（约 200-250 行），包含：
- HTML：输入框、查询按钮、结果展示区域（默认隐藏）
- CSS：简洁布局、加载动画（旋转圆圈）、错误提示样式（红色边框）
- JavaScript：
  - `fetchWeather(city)` 函数：拼接 API URL、发送请求、解析 JSON
  - `renderWeather(data)` 函数：DOM 操作展示结果
  - `showError(msg)` 函数：错误处理
  - `showLoading()` / `hideLoading()` 函数：加载状态切换
  - 天气状况映射（晴天→☀️、雨天→🌧️ 等）

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| API Key 泄露 | 学生把 Key 直接写在代码里并提交到 Git | 强调"环境变量"概念：将 Key 放在单独配置文件（如 config.js）并加入 .gitignore |
| CORS 报错 | 浏览器控制台显示跨域错误 | 解释浏览器安全策略，提供两种方案：① 使用 Trae 内置代理；② 用 Node.js 写简单中转服务器 |
| 中文城市名乱码 | 查询"北京"时 URL 编码错误 | 提醒学生使用 `encodeURIComponent(city)` 处理中文参数 |
| API 调用超限 | QWeather 免费版有每日限额 | 建议学生申请自己的 Key，不要共用；或添加本地缓存（同一城市 10 分钟内不重复请求） |

**讲师应答脚本**：

- **学生问"和风天气注册不了怎么办"**："可以用其他免费天气 API 替代，如 Open-Meteo（无需 Key）或心知天气。将 API 地址替换即可，逻辑相同。"
- **学生问"怎么显示未来 3 天预报"**："在 Chat 模式中选中 JavaScript 部分，输入'添加未来 3 天天气预报展示'。Trae 会调用 QWeather 的 3 天预报接口并扩展 render 函数。"

### 案例 3-3 【挑战】智慧农业数据可视化大屏

**目标**：让学生综合运用 Trae Builder + Chat 模式、ECharts 图表、多文件项目结构。

**完整提示词（可复制给学生，在 Trae Builder 中使用）**：

```
请帮我创建一个"智慧农业数据可视化大屏"，要求：

整体布局：
1. 顶部：标题"智慧农业监测大屏" + 当前时间（实时更新）
2. 左侧（占 30%）：环境数据卡片区（温度、湿度、光照、土壤 pH）
3. 中间（占 40%）：农场地图/示意图 + 设备状态
4. 右侧（占 30%）：图表区（近 7 天温度曲线 + 作物生长进度）
5. 底部：滚动消息栏（模拟预警信息）

技术栈：
- HTML + CSS + JavaScript
- 使用 ECharts 绘制图表（通过 CDN 引入：https://cdn.jsdelivr.net/npm/echarts@5.4.3/dist/echarts.min.js）
- 使用模拟数据（JSON 格式内嵌在 JS 中）

具体功能：
1. 环境数据卡片：每个卡片显示当前数值、与昨日对比的涨跌箭头、状态标签（正常/警告/危险）
2. 农场地图：用 CSS 画一个简单的网格图，标注温室、灌溉区、仓库，不同颜色表示状态（绿色正常、黄色警告、红色故障）
3. 温度曲线图：ECharts 折线图，X 轴为日期，Y 轴为温度，带平滑曲线和区域填充
4. 作物生长进度：ECharts 柱状图，展示 4 种作物（小麦、玉米、水稻、大豆）的当前生长百分比
5. 预警消息栏：3 条消息循环滚动，如"【警告】温室 A 温度超过 35°C"、"【提醒】灌溉区 B 需补水"

视觉要求：
- 深色背景（#0f1419），科技感
- 卡片带边框发光效果（box-shadow）
- 数字使用等宽字体（font-family: 'Courier New', monospace）
- 响应式：最小支持 1280x720，最大支持 1920x1080

请生成多文件项目结构（index.html + css/style.css + js/data.js + js/charts.js），并确保在 Trae 中可以直接运行。
```

**预期输出**：

Trae Builder 生成一个多文件项目：

```
project/
├── index.html          # 页面结构和布局
├── css/
│   └── style.css       # 深色主题、卡片样式、布局网格、响应式媒体查询
├── js/
│   ├── data.js         # 模拟数据（环境数据、温度历史、作物进度、预警消息）
│   └── charts.js       # ECharts 初始化、折线图配置、柱状图配置
```

关键代码特征：
- `index.html`：使用 CSS Grid 布局（`grid-template-columns: 30% 40% 30%`），引入 ECharts CDN
- `style.css`：深色背景、`.card` 类带 `box-shadow: 0 0 15px rgba(76, 175, 80, 0.3)`、`.status-normal/warning/danger` 状态色
- `data.js`：包含 `environmentData`、`temperatureHistory`、`cropProgress`、`alerts` 四个对象
- `charts.js`：`initTemperatureChart()` 和 `initCropChart()` 函数，带 `window.addEventListener('resize', chart.resize)`

**学生常见陷阱**：

| 陷阱 | 表现 | 讲师应对 |
|------|------|----------|
| ECharts 未加载 | 图表区域空白 | 检查 CDN 链接是否可访问（需联网）。如网络受限，可下载 echarts.min.js 到本地 js 文件夹并修改引入路径 |
| 布局错乱 | 窗口缩小时元素重叠 | 提醒学生检查 CSS 媒体查询是否生成正确，或在 Chat 模式中输入"修复 1366x768 分辨率下的布局重叠问题" |
| 数据不更新 | 时间或数值是静态的 | 引导学生找到 data.js 中的数据对象，理解这是"模拟数据"。如需动态效果，可在 Chat 模式中添加 `setInterval` 随机微调数值 |
| 多文件路径错误 | 浏览器打开后 CSS/JS 加载失败（404） | 强调相对路径的重要性：CSS 路径是 `css/style.css`（不是 `/css/style.css`），确保文件结构和引用一致 |

**讲师应答脚本**：

- **学生问"怎么接入真实传感器数据"**："真实项目需要后端接口（Node.js + 数据库）接收传感器推送。当前是前端演示版本，真实数据替换 data.js 中的模拟对象即可。"
- **学生问"能不能部署到网上让同学看"**："可以。最简单的方式：① 用 GitHub Pages（免费）；② 用 Vercel/Netlify（免费，需注册）。课程结束后可专门讲部署。"
- **学生问"图表太复杂看不懂"**："ECharts 配置是'声明式'的，每个属性对应图表的一个视觉元素。建议先修改 data.js 中的数值，观察图表变化，再逐步理解配置项含义。"

## 附录：案例使用指南

### 讲师授课节奏建议

| 时间 | 动作 |
|------|------|
| 案例开始前 5min | 说明案例目标、展示最终效果截图/GIF |
| 案例进行中 | 巡回指导，优先帮助"基础"案例有困难的学生，不要直接给答案 |
| 案例结束前 10min | 收集 2-3 个典型问题，统一讲解 |
| 案例结束后 | 展示优秀学员作品，点评亮点 |

### 难度调整建议

- **基础班（非计算机专业为主）**：Day 1 只做案例 1-1，Day 2 只做案例 2-1，Day 3 只做案例 3-1。确保全员通关。
- **标准班（混合专业）**：按手册执行，基础+进阶必做，挑战选做。
- **提高班（计算机相关专业）**：基础案例作为热身（30 分钟），重点放在进阶和挑战案例，鼓励挑战案例的扩展（如接入真实 API、添加后端）。

### 应急备案

| 场景 | 应对 |
|------|------|
| 网络中断，QClaw/Trae 无法联网 | 切换到离线案例：Day 1 改为"用 QClaw 本地知识库问答"；Day 3 改为"纯静态页面开发"（不涉及 API） |
