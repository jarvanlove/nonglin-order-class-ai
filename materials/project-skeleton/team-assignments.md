# 农林政策信息聚合站 — 六组模块划分与任务分配

> **适用课程**：AI 智能体工具实训  
> **分组规则**：6 组 × 5 人，每组至少 2 名计算机专业学生  
> **核心原则**：工作量均等、接口统一、独立并行、最后整合

---

## 一、六组总览

| 组别 | 负责维度 | 对应数据文件 | 技术难度 | 适合专业背景 |
|------|---------|-------------|---------|-------------|
| **第1组** | 种植政策 | `data/team1-planting.json` | 简单 | 农学、园艺、植保 |
| **第2组** | 养殖政策 | `data/team2-breeding.json` | 简单 | 动物科学、畜牧 |
| **第3组** | 补贴政策 | `data/team3-subsidy.json` | 中等 | 农业经济、管理 |
| **第4组** | 环保政策 | `data/team4-environment.json` | 中等 | 环境科学、生态 |
| **第5组** | 林业政策 | `data/team5-forestry.json` | 简单 | 林学、园林 |
| **第6组** | 科技政策 | `data/team6-scitech.json` | 中等 | 计算机、农业工程 |

---

## 二、每组内部角色分工（5人）

每组 5 人，建议按以下角色分配：

| 角色 | 人数 | 职责 | 适合人选 |
|------|------|------|---------|
| **组长** | 1人 | 协调进度、汇总成果、对接老师 | 有组织能力的同学 |
| **QClaw 采集员** | 1-2人 | 负责信息搜集、Prompt 编写、数据导出 | 善于表达、会写指令的同学 |
| **WorkBuddy 处理员** | 1人 | 负责数据清洗、格式转换、报告生成 | 细心、会用 Excel 的同学 |
| **Trae 开发员** | 1-2人 | 负责页面调整、数据对接、样式微调 | 计算机专业同学优先 |

## 三、各组详细任务说明

---

### 第1组：种植政策

#### 3.1.1 负责模块
- 搜集与**农作物种植**相关的国家和地方政策
- 包括：粮食作物、经济作物、良种推广、种植技术、耕地保护等

#### 3.1.2 数据接口定义

```json
{
  "id": "planting-001",
  "title": "政策标题",
  "category": "种植政策",
  "subCategory": "粮食作物/经济作物/良种推广/种植技术/耕地保护",
  "source": "发布机构",
  "publishDate": "2026-XX-XX",
  "region": "全国/某省/某市",
  "summary": "摘要",
  "keyPoints": ["要点1", "要点2", "要点3"],
  "amount": "如适用，补贴金额",
  "targetGroup": "适用对象",
  "url": "原文链接",
  "tags": ["水稻", "小麦", "玉米", "大豆"],
  "status": "现行有效/已废止/征求意见稿"
}
```

#### 3.1.3 QClaw 采集任务

| 项目 | 内容 |
|------|------|
| **采集网站** | 农业农村部官网、各省农业农村厅官网、中国政府网、农民日报 |
| **关键词** | "种植政策" "粮食补贴" "良种推广" "耕地保护" "水稻政策" "小麦政策" "玉米政策" |
| **采集数量** | 至少 10 条 |
| **输出格式** | JSON，字段见上方接口定义 |
| **Prompt 模板** | `请搜索2025-2026年关于[关键词]的国家或地方政策，返回标题、发布机构、日期、摘要、原文链接` |

#### 3.1.4 WorkBuddy 清洗任务

| 项目 | 内容 |
|------|------|
| **输入** | QClaw 导出的原始 JSON/Excel |
| **清洗规则** | 1. 去重（按标题） 2. 统一日期格式 3. 补充缺失字段（无则填"暂无"） 4. 按 subCategory 分类 |
| **输出格式** | `data/team1-planting.json`（标准 JSON 数组） |
| **额外产出** | 种植政策分类统计表（Excel） |

#### 3.1.5 Trae 开发任务

| 项目 | 内容 |
|------|------|
| **修改文件** | `data/team1-planting.json`（替换数据）、`about.html`（团队信息） |
| **实现功能** | 1. 确保种植政策数据能在 `list.html?category=planting` 正确显示 2. 首页种植政策入口卡片显示本组数据条数 |
| **可选加分** | 在首页添加"种植政策趋势"小图表 |

#### 3.1.6 验收标准

- [ ] QClaw 采集到至少 10 条种植政策
- [ ] 数据格式符合接口契约，无缺失必填字段
- [ ] `list.html?category=planting` 能正确展示本组数据
- [ ] 首页种植政策卡片显示正确的数据条数
- [ ] 团队信息已更新到 `about.html`
