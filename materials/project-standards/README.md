# 项目规范文档

本文档集合用于指导学生在 5 天 Vibe Coding 训练营中完成全栈项目。

## 规范清单

| 序号 | 文档 | 说明 |
|------|------|------|
| 01 | [环境安装指南](01-environment-setup.md) | Node.js / Git / Python / PostgreSQL / Claude Code CLI 安装与验证 |
| 02 | [项目架子规范](02-project-scaffold.md) | React + Vite + FastAPI 项目目录结构与初始化 |
| 03 | [数据库设计规范](03-database-design.md) | PostgreSQL 表设计、命名规则、字段类型建议 |
| 04 | [API 接口规范](04-api-design.md) | RESTful 接口设计、请求/响应格式、状态码规范 |
| 05 | [编码规范](05-coding-standard.md) | TypeScript / React 前端规范 + Python / FastAPI 后端规范 |
| 06 | [开发流程规范](06-development-workflow.md) | 开发 / 测试 / 对接 / 部署全流程 |
| 07 | [Claude Code CLI 配置指南](07-claude-code-guide.md) | 安装后配置、Skill / MCP / 常用命令、调教技巧 |
| 08 | [Hermes Agent 配置指南](08-hermes-agent-guide.md) | 安装后配置、Cloud API 连接、工具注册、调试技巧 |
| 09 | [标准提示词模板](09-prompt-templates.md) | 各阶段（前端/后端/数据库/Agent）的标准提示词 |

## 使用方式

1. **开课前**：学生按 `01-environment-setup.md` 完成环境安装
2. **Day 2**：讲师发布 `02-project-scaffold.md`，学生搭建项目架子
3. **Day 3**：参考 `03-database-design.md` 和 `04-api-design.md` 设计数据层
4. **全程**：参考 `05-coding-standard.md` 保持代码风格一致
5. **Day 4**：参考 `07-claude-code-guide.md` 和 `08-hermes-agent-guide.md` 进阶配置
6. **全程**：参考 `09-prompt-templates.md` 中的提示词模板，复制到 Claude Code CLI 中使用
