## 三、技术方案

### 3.1 整体架构
（请用文字或简易图表描述项目的整体技术架构，说明三大工具如何协作。）

```
【填写示例】
整体架构：

用户上传图片 → Trae 开发的前端页面接收图片 → 调用识别 API 返回结果
                                    ↓
              QClaw 自动搜集最新病虫害资料和防治方案 → 存入本地知识库
                                    ↓
              WorkBuddy 每周自动读取识别记录和知识库 → 生成 Word 报告
                                    ↓
              用户查看报告 / 下载 / 分享

数据流：
- QClaw 负责"信息输入"：定时搜集网络资料，更新知识库
- WorkBuddy 负责"数据处理"：清洗、汇总、生成报告
- Trae 负责"功能开发"：前端页面、后端接口、数据库操作
```

步骤 5：QClaw 每晚定时搜集最新资料，更新知识库 Markdown 文件。
步骤 6：WorkBuddy 每周一自动读取 Excel 记录，生成 Word 报告并归档。
步骤 7：用户可在前端页面查看历史记录，或下载 WorkBuddy 生成的周报。
