# 项目选题库

**适用课程**：AI 智能体工具实训  
**目标学员**：农林高校应届毕业生（计算机 + 非计算机混编）  
**工具组合**：QClaw（信息搜集）+ WorkBuddy（数据处理）+ Trae（展示开发）  
**开发周期**：4 天（Day 3-6）  

---

## 选题分类说明

| 分类 | 说明 | 适合人群 |
|------|------|---------|
| A 类：通用界面型 | 以信息展示和查询为主，界面友好，技术难度较低 | 非计算机专业占比高的组 |
| B 类：数据分析型 | 以数据处理和分析为主，需要一定的数据思维 | 计算机和非计算机均衡的组 |
| C 类：农林特色型 | 与农林领域深度结合，需要专业知识背景 | 农林专业背景强的组 |

---

## A 类：通用界面型（6 个选题）

### A1. 校园资讯聚合平台

| 属性 | 内容 |
|------|------|
| **名称** | 校园资讯聚合平台 |
| **工具组合** | QClaw + WorkBuddy + Trae |
| **难度** | 简单 |
| **核心功能** | 自动搜集学校官网、教务处、图书馆等渠道的资讯，分类整理后展示在网页上 |

**功能描述**：
1. **QClaw 模块**：配置多个采集源（学校官网、教务处、图书馆、就业中心等），定时采集最新通知和资讯
2. **WorkBuddy 模块**：对采集的资讯进行去重、分类（学术/活动/通知/就业）、提取关键信息（时间、地点、负责人）
3. **Trae 模块**：开发一个资讯展示网页，支持按分类筛选、按时间排序、关键词搜索、收藏功能

**MVP 边界**：
- P0：能采集至少 3 个来源的资讯，分类展示，支持搜索
- P1：添加收藏功能、按时间排序
- P2：定时自动更新、邮件订阅提醒

**亮点**：贴近学生生活，实用性强，展示效果好

## 选题使用说明

1. **每组选择一个选题**，不能重复
